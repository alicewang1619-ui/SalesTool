## 关键发现

### 1. 多项目/资源管理是中大型刚需，已有成熟方案
Worktile 具备成熟的多项目管理能力，通过项目集/项目组合实现资源共享，并提供全局项目统计报表多维度实时统计各项目过程数据。ONES 强调项目集，适合大型企业。
来源：[cnblogs-30款大比拼](https://www.cnblogs.com/worktile/articles/18932524)、[testerhome 选型](https://testerhome.com/topics/37026)

### 2. ⭐ 行业反复出现的痛点 = 灯塔的机会：工具太重、员工不愿意填
禅道界面复杂、上手成本高、易用性差，"可能导致员工不愿意填的现象"。这与我们标记的 **A-002 头号风险** 完全吻合——也正是差异化空间：现有产品都在"做全、做重"，没人专门做"轻量的管理层总览 + 自动预警"层。
来源：[PingCode vs 禅道对比](https://docs.pingcode.com/blog/tool/14195.html)、[35bo 对比](https://www.35bo.com/articles/188500.html)

### 3. 代码集成是研发场景标配（飞书项目/PingCode 最强）
开发可在代码提交信息里关联任务 ID，实现代码变更与任务自动关联——这给灯塔 A-002 对策一个思路：**进度可从代码/任务流自动推断，减少手填**。
来源：[segmentfault 对比](https://segmentfault.com/a/1190000047711822)

### 4. 项目健康度"红黄绿/三级预警"是行业实操共识
PMO 实操采用三级预警体系（黄/橙/红），用资源平衡图、负载图确保资源在项目间合理分配、避免过载或闲置——印证灯塔的"自动预警 + 人力负载视图"方向正确。
来源：[ONES-PMO组合挑战](https://ones.cn/blog/knowledge/pmo-5-key-challenges-successful-project-portfolio-management)

## 🔬 ONES 深度拆解（用户指定深挖）

ONES 是最接近灯塔"管理层总览"视角的竞品，对标价值最高。

**它已经做了灯塔想做的那一层**：
- **项目集（ONES Plan）**：为 PMO 提供管理者视角，多项目统一管理、及时发现风险、统筹调度资源；清晰项目层级 + 多级汇总概览 + 整体把握项目健康度。
- **资源管理（ONES Resource）**：查看项目和团队的资源利用率与负荷度，可视化成员工作规划，甘特视图做资源规划与进度跟踪。
- **工时管理**：项目工时 + 绩效评估。
- **报表/仪表盘**：多种资源报表，实时反馈项目投入。
  来源：[ONES Plan 项目集](https://ones.cn/solutions/plan)、[ONES Resource](https://ones.cn/products/resource)、[ONES Project](https://ones.cn/products/project)

**定价**：SaaS 企业版 ¥7200 起；私有部署按模块 + 人数、人/年收费，需销售报价。
  来源：[ONES 定价](https://ones.cn/pricing)、[私有部署价格](https://help.ones.pro/quick_start/cjwt/qi-ta-wen-ti/si-you-bu-shu-shen-me-jia-ge/)

**ONES 的命门（= 灯塔的赢面）**：
- 功能叠加太多（需求池、DevOps 集成…）→ 界面繁琐、新用户上手慢；**对中小/中型团队功能过剩，复杂反成效率阻碍**。
- 权限/通知缺乏动态调整，多角色信息流转易断层。
- 报表仅基础统计、第三方集成弱、数据孤岛。
  来源：[ONES 缺点分析](https://ones.com.cn/knowledge/ones-company-analysis-strengths-challenges-wise-choice)、[Zoho 对比](https://www.zoho.com.cn/projects/articles/dutieses.html)

**结论**：ONES 证明了"管理层总览 + 健康度 + 人力负载"这层**做得出来、有人买**（不是空白市场），但它把这层埋在全功能研发平台里、太重。灯塔的差异化必须靠 **把这层单独抽出来做轻 + 录入负担压到最低 + 私有部署 + 自研贴合**——150 人正是 ONES "嫌重"、灯塔"刚好"的甜区。

## 对灯塔的启示（差异化定位）
- 不跟 PingCode/ONES 拼"研发全流程功能全"——那是红海且我们已定"不做重型 Jira"
- 切口：**管理层视角的轻量多项目总览 + 自动健康度预警 + 人力负载一眼清**，把"录入负担"压到最低（自动推断 + 极简更新）
- 私有部署是国内研发团队普遍诉求，灯塔走内网部署符合主流
