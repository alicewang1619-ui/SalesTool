# 项目灯塔 · 开发版（可开工）

> 完整细节见 PRD详细版.md。本版用于程序员/AI 编码助手直接开工。

## 范围（MVP = 阶段一）
做 FR-1 ~ FR-7（P0）+ FR-8（P1）。不做需求/缺陷/测试/工作流。任务仅作进度载体。

## 技术栈建议
- 形态：Web 响应式（桌面优先）+ 手机只读查看页；同一前端按角色与屏幕显隐模块；不做独立 App。
- 架构：单体 + 关系型数据库，前后端分离。当前规模（≤200 人 / ≤50 项目）无需微服务。
- 部署：公司内网/私有服务器，容器化，数据不出公司。
- 周报提取：本地可部署中文 NLP/语义模型 + 关键词规则兜底，不调联网外部大模型 API。

## 功能需求（P0/P1）
| 编号 | 功能 | 优先级 | 要点 |
|---|---|---|---|
| FR-1 | 多项目总览看板 | P0 | 卡片/行列出在跑项目；显示进度%、健康灯、最近更新、未关闭风险数；默认按健康度排序红置顶；可按负责人/健康度/更新时间筛选；停滞项目显式空白徽标；30 项目一屏可扫读 |
| FR-2 | 项目进度 | P0 | 里程碑挂任务；任务状态未开始/进行中/已完成，≤3 点击切换；项目进度=已完成任务占比自动算；里程碑计划vs实际轻量时间线 |
| FR-3 | 周报智能提取 | P0 | 粘贴/上传周报→提取进度/风险/阻塞生成预填草稿→负责人确认或修正落库；格式不识别回退空白草稿 |
| FR-4 | 风险登记+自动预警 | P0 | 风险条目：描述/等级(高中低)/责任人/状态(待处理/处理中/已关闭)/来源；自动预警：逾期、停滞、超载→标红进预警列表 |
| FR-5 | 人力负载视图 | P0 | 按人展示参与项目数/任务数/投入占比之和；超阈值标超载；按团队或项目维度看；投入占比由负责人粗估，不要成员填工时 |
| FR-6 | 健康度规则引擎 | P0 | 预设规则（进度落后超阈值/有逾期/有高风险/停滞超天数/关键成员超载）+ 阈值可调启停；综合判定红黄绿；规则变更全量异步重算 |
| FR-7 | 角色权限+内网账号 | P0 | 自建账号；3 角色按权限矩阵控制；管理层建账号/分配角色/归属项目；关键操作写审计日志 |
| FR-8 | 预警通知+手机查看 | P1 | 站内未读预警；手机浏览器只读页：进度/预警/我的任务 |

## 数据模型（字段含义级）
- 用户 User：姓名、账号、角色、所属团队
- 项目 Project：名称、负责人、状态(在跑/归档)、整体进度、健康灯色、最近更新时间
- 里程碑 Milestone：所属项目、名称、计划完成、实际完成、进度
- 任务 Task：所属里程碑、负责人、状态、计划时间
- 风险 Risk：所属项目、描述、等级、责任人、应对状态、来源(人工/周报/自动)
- 周报 WeeklyReport：所属项目、周次、原文、提取草稿、确认状态
- 人力投入 Allocation：用户、项目、投入占比(粗估)、生效周期
- 预警 Alert：所属项目、类型(逾期/停滞/超载/高风险)、触发时间、状态
- 健康度规则 HealthRule：规则类型、阈值、是否启用
- 审计日志 AuditLog：操作人、操作类型、对象、时间

派生字段：项目进度（由任务状态实时算，不可手工覆盖）、健康灯色（规则引擎统一重算）。

## 关键流程
1. 周报驱动更新：撰写周报→上传/粘贴→系统提取草稿→确认/修正→落库→健康度重算→看板刷新→命中规则则标红预警。
2. 健康灯色状态机：绿↔黄↔红，由进度偏差/逾期/停滞/风险等级/超载驱动。

## 验收标准（可转测试）
- AC-1 管理层登录默认进总览，按健康度排序红置顶，单屏扫读 30 项目
- AC-2 上传周报后草稿提取进度与风险，确认后看板更新
- AC-3 成员 ≤3 点击切换任务状态，进度%自动更新
- AC-4 项目超停滞天数无更新→空白徽标 + 停滞预警
- AC-5 周报格式不识别→回退空白草稿不报错
- AC-6 同一任务并发提交→最后提交为准 + 留修改记录
- AC-7 提取引擎不可用→回退关键词规则或空白草稿不阻塞
- AC-8 成员看不到非参与项目；负责人不能改规则；仅管理层建账号改规则
- AC-9 桌面浏览器全功能；手机浏览器只读页可用
- AC-10 规则阈值变更后全量重算不丢历史；新增项目不影响既有看板

## 边界与异常
- 规模上限按 200 人/50 项目设计；任务数异常膨胀→项目详情分页，看板只展示汇总灯色
- 同一任务并发→最后提交为准留记录；规则变更异步重算期间看板显示上次结果
- 同周重复上传周报→以最新确认为准，历史保留
- 数据治理：统一东八区；成员 PII 仅内网流转不出公司；审计日志不可删
