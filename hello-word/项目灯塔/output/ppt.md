# 项目灯塔 · 方案汇报 PPT 大纲

> 8 段结构，内容全部取自 PRD详细版.md。约 11 页。配色：B 端冷静蓝灰（AI 自定，无设计系统）。

## P1 · 首页
- 标题：项目灯塔 Beacon
- 一句话定位：让管理层在一个看板上看清所有项目的进度、风险与人力
- 副标题：面向 150 人研发团队的轻量多项目统管工具

## P2 · 执行摘要（我们到底做了个什么）
- 一句话：把分散在表格和群聊里的 30 个项目状态，收敛成一处可信的全局视图
- 三个关键词：一眼看清 · 自动预警 · 轻到有人用
- 边界：只做总览层，不做重型研发平台

## P3 · 核心痛点场景
- 场景：150 人研发、30 个项目并行，现靠 Excel + 钉钉群 + 脑子
- 三痛：进度逐个问、风险事后才知道、人力谁忙谁闲是黑盒
- 结论句：项目越多，管理层越看不清

## P4 · 为什么这问题值 200 万
- 测算：150 人 × 30 万元/人 × 5% ≈ 225 万元/年损耗
- 最锋利一句：你 150 人的研发团队，每年有 7.5 个人的产能，悄悄浪费在"找进度、救火、撞资源"上
- 来源：三个数均由企业方确认，损耗比例取行业最保守值

## P5 · 产品设计 1：多项目总览看板（核心）
- 30 个项目红黄绿一屏看清，按健康度排序、红色置顶
- 暴露"谁没更新"——空白徽标形成督促
- 结论句：管理层 1 分钟掌握全局健康度

## P6 · 产品设计 2：进度 + 周报提取（打掉头号风险）
- 头号风险：全员协作"没人愿意填"
- 四重对策：≤3 点击 / 看板暴露空白 / 周报自动提取 / 自动预警
- 核心招：把更新从"填表"变成"贴周报 + 确认"

## P7 · 产品设计 3：风险预警 + 人力负载
- 风险双轨：人工登记 + 系统自动预警（逾期/停滞/超载标红）
- 人力负载：谁排爆谁有余量，排人前先看
- 健康度规则：预设 + 阈值可调

## P8 · 技术方案 1：架构与部署
- 单体 + 关系型数据库，前后端分离，当前规模无需微服务
- Web 响应式 + 手机只读查看页
- 内网/私有部署，数据不出公司；周报提取用本地 NLP，不联网

## P9 · 技术方案 2：数据模型与模块边界
- 核心实体：项目/里程碑/任务/风险/周报/人力投入/预警/健康度规则
- 六大模块：看板 / 周报提取 / 风险预警 / 人力负载 / 健康度引擎 / 账号权限
- 派生字段自动算，不可手工覆盖

## P10 · 步骤阶段（完整路线图）
- 阶段一 MVP：四块 + 三角色 + 内网账号
- 阶段二：Git 集成 / 钉钉飞书 SSO 与推送 / 完整甘特 / 报表导出
- 阶段三：跨项目复盘 / 工时成本 / 外部只读
