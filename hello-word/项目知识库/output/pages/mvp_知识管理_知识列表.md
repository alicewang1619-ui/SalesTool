# 页面文档 · 知识列表（首页）

- **文件**：mvp_知识管理_知识列表.html
- **阶段/模块**：MVP / 知识管理（FR-8，FR-1 入口）
- **路由**：`/`（应用首页）

## 1. 页面目标
登录后/打开即第一屏。让用户要么立刻搜（顶部大搜索框），要么扫一眼最近存了什么（卡片流）。是全站枢纽。

## 2. 用户与权限
单用户本地自用，无角色分层。

## 3. 布局结构
左侧固定导航（全部知识/标签/问答/设置）+ 顶部全局搜索框 + 右上「＋新增知识」+ 主区知识卡片流（最近添加优先）+ 来源筛选 chips。

## 4. 核心组件
- 全局搜索框（提交 → 检索结果页）
- 知识卡片：标题 / 2 行摘要 / 标签 / 来源类型 / 时间；hover 阴影提升
- 来源筛选 chips：全部/最近/公众号/AI对话/心得/文件

## 5. 数据来源
Knowledge 列表（按 created_at 倒序），含 title、summary、tags、source_type、created_at。

## 6. 状态设计
- **空库（首次）**：展示欢迎引导 + 醒目「新增第一条知识」按钮，引导去录入页。
- **加载**：卡片骨架屏。
- **无更多**：无限滚动到底提示「没有更多了」。

## 7. 交互
- 点卡片 → 知识详情页
- 顶部搜索提交 → 检索结果页
- ＋新增 → 录入页
- chips 切换 → 前端筛选列表
- 无限滚动加载更多

## 8. 验收（关联 AC）
- 卡片正确展示标题/摘要/标签/来源/时间。
- 搜索框提交跳检索结果（支撑 AC-3 入口）。
- 空库展示引导（支撑 AC-1 首次录入路径）。

## 9. 跳转关系
→ 检索结果 / 录入 / 知识详情 / 左导航各页。
