# TDD 测试设计文档：性能基线

> 给 Coding Agent 消费：按优先级从 P0 起逐条翻译成可执行测试、逐条勾绿。

---

## 1. 单元元信息

| 字段 | 内容 |
| :--- | :--- |
| 测试层 | 性能（横切） |
| 单元短码 | `PERF` |
| 对应上游产物 | PRD §8 非功能、技术方案 |
| 所属产品 / 模块 | 项目知识库 / 性能基线 |
| 覆盖范围（横切） | 检索、列表、本地模型调用 |
| 技术栈 | 以技术方案为准 |
| 复杂度 | M |

> 本地模型耗时依赖用户机器，做基线记录而非硬阈值。

## 2. 测试策略

### 2.1 测试优先级

| 优先级 | 测试维度 | 依据来源 | 理由 |
| :--- | :--- | :--- | :--- |
| P1 | 检索响应/列表流畅 | PRD §8 | 核心体验阈值 |
| P2 | embedding/模型/启动基线 | 基线、体验 | 记录回归基线 |

### 2.2 不测什么
- **不测** 本地模型生成速度的硬阈值 —— 依赖机器，只记基线。

## 4. Mock 策略

### 4.1 必须 Mock 的外部依赖
| 依赖 | Mock 方式 | 理由 | 关联用例 |
| :--- | :--- | :--- | :--- |
| 测试数据生成 | 批量生成几千条知识 | 真实规模才能测性能 | P-PERF-01/02 |

### 4.2 禁止 Mock 的内部单元
| 单元 | 理由 |
| :--- | :--- |
| 检索/渲染真实耗时 | 性能必须测真实执行时间 |

> 性能用例用真实数据量与真实执行，不可用 mock 缩短路径，否则基线无意义。

**P2 实现顺序**：`P-PERF-03 → P-PERF-04 → P-PERF-05`
