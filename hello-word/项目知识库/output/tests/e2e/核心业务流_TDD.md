## 2. 测试策略

### 2.1 测试优先级

| 优先级 | 测试维度 | 依据来源 | 理由 |
| :--- | :--- | :--- | :--- |
| P0 | 录入/检索/问答/离线全链路 | 流A/B/C、AC-1~7 | 核心闭环命脉，断一环产品就不成立 |
| P1 | 标签/删除恢复链路 | 流D/E、AC-5/8 | 辅助流程 |

### 2.2 不测什么
- **不测** 单步内部逻辑（已由单元/组件/集成层覆盖）—— E2E 只验证跨页面链路贯通。

## 4. Mock 策略

### 4.1 必须 Mock 的外部依赖
| 依赖 | Mock 方式 | 理由 | 关联用例 |
| :--- | :--- | :--- | :--- |
| 外部抓取目标 | 本地桩页面/固定 HTML | 真实公众号有反爬、不稳定 | E-FLOW-01/02 |
| 云端大模型 | 桩端点（仅非离线用例） | 避免计费、可控响应 | E-FLOW-03/04 |
| 网络（断网） | 拦截外网 | 验证离线可用 | E-FLOW-07 |

### 4.2 禁止 Mock 的内部单元
| 单元 | 理由 |
| :--- | :--- |
| 本地存储/检索/页面跳转 | E2E 必须走真实应用链路，mock 内部即失去端到端意义 |
| 本地 Ollama（离线用例） | E-FLOW-07 须用真实本地模型证明离线可用 |

> E2E 只 mock 系统外边界（外网/计费），内部全链路用真实实现。
