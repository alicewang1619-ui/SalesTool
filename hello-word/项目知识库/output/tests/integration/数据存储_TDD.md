## 2. 测试策略

### 2.1 测试优先级

| 优先级 | 测试维度 | 依据来源 | 理由 |
| :--- | :--- | :--- | :--- |
| P0 | CRUD/多对多/向量读写/删除事务一致 | §5、§3.3 | 数据一致性是检索正确的根基 |

## 4. Mock 策略

### 4.1 必须 Mock 的外部依赖
| 依赖 | Mock 方式 | 理由 | 关联用例 |
| :--- | :--- | :--- | :--- |
| 无（集成层用真实测试库） | 用临时 SQLite + 临时 LanceDB 实例 | 集成契约必须测真实存储行为 | 全部 |

### 4.2 禁止 Mock 的内部单元
| 单元 | 理由 |
| :--- | :--- |
| SQLite/LanceDB 读写 | 本层正是要验证真实存储契约，mock 即失去意义 |
| 删除事务 | 跨双库一致性是被测核心 |

> 集成层禁止 mock 数据库——用隔离的临时库实例，每个用例前后清理。
