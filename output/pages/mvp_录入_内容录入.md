# 页面文档 · 内容录入

- **文件**：mvp_录入_内容录入.html
- **阶段/模块**：MVP / 内容录入（FR-1，触发 FR-2）
- **路由**：`/new`

## 1. 页面目标
四种方式把内容丢进库，录入越轻越好——这是产品成败关键（A-003）。入库后触发自动整理。

## 2. 用户与权限
单用户本地自用。

## 3. 布局结构
顶部返回 + 四 Tab（贴链接 / 粘贴文本 / 写一条 / 上传文件），每 Tab 一个面板 + 状态区 + 保存/取消。

## 4. 核心组件
- 链接输入 + 抓取按钮
- 标题/正文输入、Markdown 编辑器
- 文件拖拽上传区（支持多选/多文件拖入，逐个解析，汇总成功与失败）
- 状态条（抓取中/抓取失败回退/解析中/整理中/成功）

## 5. 数据来源/写入
新建 Knowledge（source_type=link/paste/note/file），异步生成 embedding + 大模型打标签/摘要。

## 6. 状态设计
- **抓取中 / 解析中**：spinner 提示。
- **抓取失败（公众号反爬 A-001）**：红色提示 + 一键切到「粘贴文本」回退。
- **学术论文（含 DOI / arXiv 链接，如 Science/Nature/arXiv）**：优先经开放 API（Crossref / arXiv）取标题+作者+摘要入库，绕开付费墙/反爬。
- **其它付费/反爬站点（HTTP 401/402/403/429）且无 DOI**：服务器无法抓取，明确提示改用**浏览器剪藏扩展**（在已登录页面剪藏）或**粘贴文本**。
- **整理中**：「AI 正在打标签和摘要」。
- **草稿**：编辑器自动存草稿；离开未保存拦截。

## 7. 交互
- 切 Tab、抓取、上传、保存（→ 知识详情）、取消（→ 列表）
- 抓取失败 → 切粘贴 Tab

## 8. 验收（关联 AC）
- **AC-1**：四种方式均能入库；公众号抓取失败回退粘贴。
- **AC-2**：入库后触发自动标签+摘要（整理中→详情页可见）。

## 9. 跳转关系
→ 知识详情（保存后）/ 知识列表（取消）。
