Redis 缓存穿透与击穿解决方案

📎 来源：微信公众号原文 ↗ ·2 小时前添加
✨ AI 摘要编辑

缓存穿透指查询不存在的数据导致请求直达数据库，用布隆过滤器或空值缓存拦截；缓存击穿指热点 key 失效瞬间大量请求打到数据库，用互斥锁或逻辑过期解决。

一、缓存穿透

缓存穿透是指查询一个数据库和缓存中都不存在的数据，每次请求都会穿过缓存打到数据库。常见于恶意攻击用不存在的 id 刷接口。

解决方案有两种：① 布隆过滤器——把所有可能存在的 key 预先放入，查询前先过滤；② 空值缓存——对查不到的 key 也缓存一个空值并设短过期。

二、缓存击穿

缓存击穿是指某个热点 key 在失效的瞬间，大量并发请求同时打到数据库。解决方案：① 互斥锁——只让一个请求去重建缓存；② 逻辑过期——不设真实过期，异步更新。
