## 2. 测试策略

### 2.1 测试优先级

| 优先级 | 测试维度 | 依据来源 | 理由 |
| :--- | :--- | :--- | :--- |
| P0 | Tab切换/抓取状态/失败回退/整理反馈/保存跳转 | FR-1回退、FR-2、§6/§7 | 录入是入口、失败回退是成败关键 |
| P1 | 草稿/超大文件提示 | §6、边界 | 防丢失与边界 |
| P2 | 防重复提交 | 边界 | 稳健性 |

### 2.2 不测什么
- **不测** 实际抓取/解析逻辑 —— 由 `unit/录入解析_TDD.md` 负责。
- **不测** 入库后数据落库 —— 由 `integration/数据存储_TDD.md` 负责。

## 4. Mock 策略

### 4.1 必须 Mock 的外部依赖
| 依赖 | Mock 方式 | 理由 | 关联用例 |
| :--- | :--- | :--- | :--- |
| 抓取接口 | 返回成功/失败 | 驱动抓取中/失败回退状态 | C-ING-02/03 |
| 入库接口 | 返回成功 | 驱动整理中/保存跳转 | C-ING-04/07 |

### 4.2 禁止 Mock 的内部单元
| 单元 | 理由 |
| :--- | :--- |
| Tab 切换组件 | 被测对象本身 |
| 草稿自动保存逻辑 | 被测对象本身 |

> Mock 抓取失败必须触发真实回退 UI，不能只返错码不验证回退入口出现。
