## 2. 测试策略

### 2.1 测试优先级

| 优先级 | 测试维度 | 依据来源 | 理由 |
| :--- | :--- | :--- | :--- |
| P0 | 核心渲染/相关推荐/删除确认 | 知识详情§4、知识编辑§6 | 阅读与关联是核心、删除需防误删 |

- **不测** 删除的持久化与回收站存储 —— 由 `integration/数据存储_TDD.md` 负责。

## 4. Mock 策略

### 4.1 必须 Mock 的外部依赖
| 依赖 | Mock 方式 | 理由 | 关联用例 |
| :--- | :--- | :--- | :--- |
| 知识数据查询 API | 返回固定列表/空列表/相关推荐集 | 隔离后端，专注渲染 | C-KM-01/02/06/07 |
| 删除/恢复接口 | 返回成功 | 隔离持久层 | C-KM-09 |

### 4.2 禁止 Mock 的内部单元
| 单元 | 理由 |
| :--- | :--- |
| 列表渲染/筛选组件 | 被测对象本身 |
| 删除确认弹窗交互 | 被测对象本身 |

> Mock 数据必须含完整卡片字段（标题/摘要/标签/来源/时间），缺字段会让渲染断言假绿。
