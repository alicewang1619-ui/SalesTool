## 2. 测试策略

### 2.1 测试优先级

| 优先级 | 测试维度 | 依据来源 | 理由 |
| :--- | :--- | :--- | :--- |
| P0 | 隐私零外发/外发提示/XSS/注入/Key脱敏 | §6、门禁硬门 | 隐私与注入是不可降级硬门 |
| P1 | 文件解析安全/备份校验 | 文件解析、AC-8 | 防越权读盘与恶意导入 |
| P2 | 本地密码锁 | §9 后置 | 后置功能 |

### 2.2 不测什么
- **不测** 传统 RBAC 越权 —— 本产品单用户本地，无此面。
- **不测** 登录会话 —— MVP 无登录。

## 4. Mock 策略

### 4.1 必须 Mock 的外部依赖
| 依赖 | Mock 方式 | 理由 | 关联用例 |
| :--- | :--- | :--- | :--- |
| 外网请求监听 | 拦截并记录所有出站请求 | 验证零外发/最小外发 | S-SEC-01/02 |
| 日志/导出输出 | 捕获输出内容做断言 | 验证 Key 不泄露 | S-SEC-05 |

### 4.2 禁止 Mock 的内部单元
| 单元 | 理由 |
| :--- | :--- |
| 正文渲染清洗 | XSS 防护是被测核心，mock 即假绿 |
| SQL 参数化层 | 注入防护是被测核心 |

> 安全用例尤其禁止"只断言不触发"——XSS/注入必须用真实恶意载荷驱动真实渲染/查询。
