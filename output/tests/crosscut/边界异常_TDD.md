## 2. 测试策略

### 2.1 测试优先级

| 优先级 | 测试维度 | 依据来源 | 理由 |
| :--- | :--- | :--- | :--- |
| P0 | 失败态回退/模型不可用/API异常/空库 | 各页§6、门禁硬门 | 本产品失败态极多，一崩就废 |
| P1 | 超长/超大/并发/非法状态/空query | 边界 | 健壮性 |
| P2 | 恢复冲突 | 非法状态 | 稳健性 |

### 2.2 不测什么
- **不测** 正常路径功能 —— 由各功能层负责，本层只测异常与边界。

## 4. Mock 策略

### 4.1 必须 Mock 的外部依赖
| 依赖 | Mock 方式 | 理由 | 关联用例 |
| :--- | :--- | :--- | :--- |
| 抓取接口 | 返回失败/超时 | 驱动回退分支 | B-BND-01 |
| 本地/云端模型 | 模拟未启动/Key失效/超时 | 驱动失败态 | B-BND-02/03 |
| 数据源 | 返回空集 | 驱动空态 | B-BND-04 |

### 4.2 禁止 Mock 的内部单元
| 单元 | 理由 |
| :--- | :--- |
| 回退逻辑/空态渲染 | 被测对象本身 |
| 防重提交/截断逻辑 | 被测对象本身 |

> 异常用例必须真实触发异常路径（真超时/真空集/真超长），不能只断言文案存在。
