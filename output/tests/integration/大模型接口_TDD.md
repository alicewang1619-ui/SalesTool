## 2. 测试策略

### 2.1 测试优先级

| 优先级 | 测试维度 | 依据来源 | 理由 |
| :--- | :--- | :--- | :--- |
| P0 | schema一致/可切换/离线可用/向量维度/流水线贯通 | FR-7、AC-7 | 本地云端可切换+离线可用是硬门 |
| P1 | 超时重试/最小外发 | 边界、安全 | 失败态与隐私 |

### 2.2 不测什么
- **不测** 模型生成内容质量 —— 仅测接口契约与可切换性。
- **不测** UI 模型切换 —— 由 `component/设置_TDD.md` 负责。

## 4. Mock 策略

### 4.1 必须 Mock 的外部依赖
| 依赖 | Mock 方式 | 理由 | 关联用例 |
| :--- | :--- | :--- | :--- |
| Ollama 本地服务 | 桩服务返回约定 schema/超时 | 真实模型慢、CI 不可控 | I-LLM-01/03/05 |
| 云端 API 端点 | 桩返回 OpenAI 兼容响应 | 避免真实计费/网络 | I-LLM-02/07 |
| 网络层（断网模拟） | 拦截外网请求 | 验证离线可用 | I-LLM-03 |

### 4.2 禁止 Mock 的内部单元
| 单元 | 理由 |
| :--- | :--- |
| 本地/云端适配切换层 | 被测对象本身，可切换性是核心 |
| 录入流水线编排 | 端到端贯通是被测核心 |

> 两端 mock 必须返回**同构** schema，否则无法验证"切换不破坏"。断网用例须真实拦截外网，证明零外发。
