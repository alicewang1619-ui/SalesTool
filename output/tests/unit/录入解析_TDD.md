## 4. Mock 策略

> 只 mock 外部边界，不 mock 内部单元。**测真实行为，不测 mock 行为。**

### 4.1 必须 Mock 的外部依赖
| 依赖 | Mock 方式 | 理由 | 关联用例 |
| :--- | :--- | :--- | :--- |
| HTTP 抓取请求 | 返回固定 HTML / 非200 / 超时 | 真实公众号/网页不稳定、有反爬 | U-ING-01/02/03 |
| 文件系统读取 | 提供固定测试文件（pdf/docx/md） | 隔离真实文件依赖 | U-ING-04/05/06/07 |

### 4.2 禁止 Mock 的内部单元
| 单元 | 理由 |
| :--- | :--- |
| 正文提取算法 | 被测对象本身，mock 它等于没测 |
| 脚本清洗逻辑 | XSS 防护核心，必须测真实清洗 |

> Mock 的 HTML/文件必须镜像真实结构（含噪音节点），不能只放当前用例需要的干净片段。

## 5. 实现顺序建议（Coding Agent 回归证明协议）

> 每条用例走红-绿-重构 + 回归证明四步：
> 1. 写测试 → 跑 → **确认失败**（失败方式与"预期失败原因"一致）
> 2. 写最小实现 → 跑 → **确认通过**
> 3. **回退实现** → 跑 → **确认再次失败**（证明不是假绿灯）
> 4. 恢复实现 → 跑 → **确认通过**

**P0 实现顺序**：`U-ING-01 → U-ING-02 → U-ING-03`

**P1 实现顺序**：`U-ING-08 → U-ING-04 → U-ING-05 → U-ING-06`
