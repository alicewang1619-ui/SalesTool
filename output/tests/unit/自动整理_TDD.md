## 2. 测试策略

### 2.1 测试优先级

| 优先级 | 测试维度 | 依据来源 | 理由 |
| :--- | :--- | :--- | :--- |
| P0 | 标签/摘要生成与落库 | FR-2 验收 | 自动整理是"少录入"价值的核心 |
| P1 | 异常容错 / 标签去重 | 边界、数据模型§5 | 防脏数据与重复标签 |
| P2 | 超短内容处理 | 边界 | 稳健性 |

### 2.2 不测什么
- **不测** 大模型生成内容的"好不好"（主观质量）—— 仅测结构正确与落库。
- **不测** embedding —— 由 `检索与RAG_TDD.md` 负责。

## 4. Mock 策略

### 4.1 必须 Mock 的外部依赖
| 依赖 | Mock 方式 | 理由 | 关联用例 |
| :--- | :--- | :--- | :--- |
| 大模型生成调用 | 返回正常 JSON / 畸形 JSON / 空 | 真实模型慢且不确定，需构造异常分支 | U-AT-01/02/03/05 |
| 标签存储 | 内存/测试库 | 隔离持久层 | U-AT-04 |

### 4.2 禁止 Mock 的内部单元
| 单元 | 理由 |
| :--- | :--- |
| 整理结果解析逻辑 | 被测对象本身 |
| 标签去重逻辑 | 被测对象本身 |

> Mock 的模型响应必须镜像真实字段结构（标签数组 + 摘要），畸形响应要真实模拟（缺字段/非法 JSON）。
