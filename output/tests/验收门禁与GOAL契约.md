## 5. Coding Agent 消费协议（/go GOAL 模式）
对 测试索引.md 中每条用例（按层顺序：单元→集成→组件→E2E→横切）：

1. **红**：先写出该用例的可执行测试，运行确认它**失败**，且失败方式与该用例「预期失败原因」**对得上**（对不上说明测错了，回查）。
2. **绿**：写**最小实现**让该测试通过，不过度实现。
3. **重构**：在保持该测试及已绿测试全绿前提下整理代码。
4. **回归**：每转绿一条，重跑全部已绿用例，证明无回归，再在 测试索引.md 把该 ID 标 🟢。

技术栈按 [技术方案.md](../技术方案.md)：Tauri 2 + React 19 + TypeScript + SQLite + LanceDB + Ollama/云端 API；测试框架自选与之匹配者（如 Vitest + Testing Library + Playwright）。
