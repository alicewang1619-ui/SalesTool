## 2 各层测试重点

### 单元层（unit）来源：技术方案 §3/§5、PRD §3
- 正文抓取/解析：网页正文提取、公众号适配、抓取失败判定。
- 文件解析：PDF/Word/Markdown 文本抽取。
- 文本分块策略：长文切块边界正确、不丢内容。
- embedding 调度：入库/更新触发重建、删除级联清向量。
- 检索排序：相似度计算与 Top-K 排序、相关度分数映射。
- RAG 上下文拼装：Top-K 选取、来源 knowledge_id 收集、超长截断。
- 自动整理：标签/摘要结果解析与落库。

### 组件层（component）来源：pages/*.md（8 页）
- 每页渲染 + 关键交互 + 状态切换：知识列表（卡片/筛选/无限滚动/空库引导）、检索结果（卡片/高亮/无结果空态）、录入（四 Tab 切换/抓取状态/失败回退/草稿）、知识详情（摘要可编辑/相关推荐/标签跳转）、知识编辑（表单/删除二次确认弹窗）、标签浏览（标签云/聚合）、问答（消息流/思考态/来源点击）、设置（模型切换/备份/回收站）。

### 集成契约层（integration）来源：技术方案 §2/§3/§4
- 应用核心 ↔ SQLite：Knowledge/Tag/KnowledgeTag CRUD 与事务一致性。
- 应用核心 ↔ LanceDB：向量写入/检索/删除契约。
- 大模型调用接口契约：本地 Ollama 与云端 OpenAI 兼容 API 的请求/响应 schema 一致（可切换不破坏）。
- 录入流水线集成：录入→解析→整理→embedding→入库串起来端到端在数据层正确。

### E2E 业务流层（e2e）来源：设计蓝图 §2 站点地图 + PRD 场景
- 流 A：贴链接录入 → 自动整理 → 在列表看到 → 详情可读（含抓取失败回退粘贴分支）。
- 流 B：语义检索 → 结果卡片 → 进详情 → 看相关推荐 → 跳另一条。
- 流 C：问答提问 → 得到带来源答案 → 点来源进详情。
- 流 D：标签浏览 → 聚合 → 详情。
- 流 E：编辑 → 删除 → 回收站恢复。

### 安全鉴权层（crosscut/安全鉴权）来源：技术方案 §6
> 本项目单用户本地、无登录无 RBAC，传统越权不适用；重点转向隐私与解析安全：
- 隐私：使用云端大模型时，仅发送必要上下文、UI 明确提示；本地模式数据零外发（断网可用）。
- 文件解析安全：恶意/超大/畸形 PDF·Word 不致崩溃或越权读本地文件。
- 注入：录入内容/检索 query 中的特殊字符不破坏 SQL/向量查询；正文里的脚本在渲染时不执行（XSS）。
- API Key 本地存储不明文泄露到日志。
- 跨生效面：录入页、检索框、问答框、设置页（API Key）。

### 边界异常层（crosscut/边界异常）来源：pages/*.md 状态 + 技术方案 §8 风险
> 本项目权重最高的横切层：
- 公众号抓取失败/超时 → 回退粘贴，不卡死。
- 本地大模型未启动/不可用 → 明确提示去设置/切云端，不黑屏。
- 云端 API Key 失效/网络异常 → 报错可重试。
- 空库：检索/问答/列表/标签的空态正确。
- 超大输入：超长正文/超大文件解析的截断或拒绝。
- 并发：同时录入多条、录入中重复提交。
- 非法状态：删除后访问详情、恢复已存在项。
- 跨生效面：录入、检索、问答、列表、标签、设置。

### 无障碍层（crosscut/无障碍）来源：pages/*.md
- 键盘可达（搜索、导航、录入、问答输入）、焦点可见（已用 focus ring）、语义化标签、对比度达标、图标按钮有可读名称。

### 性能层（crosscut/性能）来源：PRD §8、技术方案
- 几千条规模语义检索响应秒级（设基线阈值）。
- 列表无限滚动流畅、大量卡片不卡。
- 本地大模型首响应时间记录基线（依赖机器，做基线非硬阈值）。

## 3 验收门禁（= Coding Agent /go GOAL 终止条件）
- **完成定义**：全部 P0 + P1 用例转绿 = 交付；P2 尽力。
- **覆盖率目标**：核心模块（抓取/解析/检索/RAG）>90%；基础设施（存储/接口）>80%；UI 胶水 >60%。
- **安全层特例**：隐私（云端数据外发提示/本地零外发）+ XSS/注入相关 P0 用例零失败，不可降级。
- **边界异常**：失败态回退类（抓取失败回退、模型不可用提示）列为 P0。
- **变异测试**：MVP 阶段不做（可后置）。
