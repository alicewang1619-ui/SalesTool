# 设计决策蓝图（已与用户确认方向，采用 AI 推荐）

> 项目：项目知识库（本地优先个人 AI 知识库）。上游 PRD：output/PRD详细版.md。
> 推进方式：用户授权采用 AI 推荐项，一次性沉淀本蓝图，总确认后进入静默批量产出。

## 1 盘子
- **端形态**：电脑网页端（桌面浏览器 / 本地应用窗口），最小宽度约 1280px。
- **响应式**：桌面优先，移动适配后置（PRD 阶段三才做手机端）。
- **首屏/工作台聚合**：醒目全局搜索框 + 知识列表（最近添加优先）+ 快捷录入入口 + 常用标签。不放统计图表（自用工具，非管理后台）。

## 2 信息架构 & 站点地图
- **导航模式**：左侧竖排主导航 + 顶部常驻全局搜索框。
- **站点地图（调用关系）**：
```text
左侧导航：全部知识 / 标签 / 问答 / 设置
顶部：全局搜索框（任何页面可搜）

全部知识(首页/列表) ──点条目──> 知识详情 ──> 编辑 / 删除(二次确认)
                   └─顶部搜索──> 检索结果页(列表卡片) ──点条目──> 知识详情
录入(右上角"+新增"打开录入页/弹层：链接/粘贴/写/上传四 Tab)
标签 ──点标签──> 该标签知识列表 ──> 知识详情
问答页(对话式：答案 + 可点击来源 ──> 知识详情)
设置页(大模型配置：本地 Ollama / 云端 API、数据导出备份)
知识详情页 ──底部/右侧──> 相关推荐 ──> 另一条知识详情
```

## 3 角色 & 权限
- **单用户、本地自用**：第一版无多角色、无权限分层。
- **登录**：本地单机，默认免登录；可选本地密码锁（后置，不进 MVP 页面）。

## 4 各功能页面拆解（含空/载/错状态）
- **FR-1 录入** → 录入页/弹层（四 Tab：贴链接 / 粘贴 / 编辑器写 / 上传文件）。状态：抓取中 / 抓取失败→回退粘贴 / 文件解析中 / 解析失败。
- **FR-2 自动整理** → 不单独成页；体现在录入后"整理中"状态 + 知识详情页可编辑的标签/摘要。
- **FR-3 语义检索** → 检索结果页（列表卡片：标题/命中片段/标签/来源/相关度）。状态：搜索中 / 无结果空态。
- **FR-4 相关推荐** → 知识详情页内的"相关知识"模块（≥3 条）。空态：内容不足时提示。
- **FR-5 标签聚合** → 标签浏览页（标签云/列表）→ 点击进该标签知识列表。空态：暂无标签。
- **FR-6 RAG 问答** → 问答页（对话式：答案文本 + 可点击来源列表）。状态：思考中 / 无相关知识 / 大模型不可用。
- **FR-7 大模型底座** → 设置页（本地/云端切换、API Key、模型选择）。状态：本地模型未就绪引导 / 连接失败。
- **FR-8 知识管理** → 知识列表（全部知识）+ 知识详情 + 编辑页 + 删除二次确认。空态：库为空的首次引导。
- **隐性页面**：空库首次引导/欢迎页、大模型未配置引导、设置页内数据导出备份区。

## 5 通用交互范式
- **列表**：无限滚动（知识流自然），支持搜索/标签筛选/排序（时间/相关度），批量操作（删除/打标签），可导出。
- **表单（录入）**：实时轻校验、自动存草稿、离开未保存拦截。
- **反馈**：轻量 Toast（成功/失败）+ 危险操作（删除）弹窗二次确认 + 回收站/可撤销。

## 6 视觉风格锚点
- **明暗**：浅色为主（深色后置）。
- **风格定调**：Linear 式科技 SaaS 的冷静克制 + 强阅读友好（长文沉浸）。
- **设计 Category**：效率/知识生产力工具。
- **情绪基调**：安静、专注、高级感，视觉噪音降到最低。
- **交互哲学**：统一「阴影提升」（subtle elevation），克制微动效，不混用色彩响应/物理位移。
- **字体气质**：无衬线、中文友好（思源黑体/系统字体栈），正文阅读舒适、行距宽松；标题略具性格。
- **设计签名**：单一温和强调色（靛蓝系）+ 大量留白 + 卡片化知识条目 + 柔和的"相关性"视觉提示。

## 7 数据与集成
- **核心实体**：Knowledge / Tag / KnowledgeTag / Embedding（见 PRD §5）。
- **外部对接**：公众号/网页正文抓取；大模型（本地 Ollama + 云端 API）。
- **数据访问**：本地应用，前端经本地服务/IPC 访问本地 SQLite + 本地向量库。
- **同步策略**：MVP 无云同步，数据全本地。

## 8 技术栈 & 架构 & 部署（PRD §6 建议，待技术方案命令实测版本）
- **形态**：Tauri（推荐，轻、安装包小）或 Electron。
- **前端**：React + TypeScript + Tailwind/shadcn。
- **后端逻辑**：Tauri→Rust；或本地服务方案 Python(FastAPI)/Node。
- **存储**：SQLite（结构化）+ LanceDB/Chroma（向量），全本地文件。
- **大模型**：Ollama 本地优先 + 云端 OpenAI 兼容 API 可配置（云端选国内可达的，如 DeepSeek/通义/智谱）。
- **部署**：本地单机，打包为桌面安装包/一键启动，无服务器。
- **国内可达**：本地优先天然规避；云端 API 默认推荐国内可稳定访问的服务。

## 9 安全 & 运维
- **认证**：本地单机默认免登录；可选本地密码锁（后置）。
- **脱敏/隐私**：默认数据不出本机；用云端大模型时仅发必要上下文并明确提示。
- **备份**：本地数据一键导出备份（PRD AC-8）。
- **审计日志**：个人自用，轻量或不做。
- **操作可逆**：删除走二次确认 + 回收站/可撤销，防误删。
