# 场景锚点（V1）· 项目知识库

**产品名（暂定）**：项目知识库（个人 AI 知识库）

**一句话定位**：一个本地优先的个人 AI 知识库工具——把每天散落多来源的信息整理沉淀，用大模型实现语义检索与知识关联，需要时一问就找到。

**核心用户**：
- 第一版：Zongmin 本人（研发技术岗，日均接触大量信息）
- 终极：销售 / 管理 / 研发 / 工程师等各类岗位的个人（人人各自管理自己的知识库）
- **不是谁**：不是公司统一文档中心，不是团队内容共享平台，不是多租户 SaaS

**核心场景**：每天从微信公众号、各类文章、与 AI 的对话记录、个人心得里获取信息 → 顺手丢进知识库 → 系统自动整理/打标签/找关联 → 过段时间需要某个资料时，用大白话一问就能精准找回，并看到相关的其他知识。

**核心痛点**：信息来源多且散（公众号、文章、AI 对话、心得各在一处），看过就忘、需要时找不回；普通收藏夹只能堆、不能"问"、更不能看出知识之间的联系。

**核心能力**（用户明确要求，全要）：
1. 检索 —— 语义检索，用大白话按意思找
2. 知识关联 —— 相关推荐 + 标签聚合（不做手动双链/图谱）
3. 库内问答 —— 基于个人库的 RAG 问答
4. 自动整理 —— 自动打标签 / 摘要 / 归类

**录入方式**（全要）：①贴链接自动抓正文 ②复制粘贴文本 ③工具内编辑器直接写 ④上传文件（PDF/Word/MD）

**大模型策略**：本地大模型优先，公开（云端）大模型可选；local-first。

**数据存储**：本地电脑为主；数据不强制上云。

**成功定义（待阶段2量化）**：需要某条知识时，能在数秒内用自然语言问答方式精准找回，且能看到相关知识——替代"翻收藏夹翻聊天记录找不到"的现状。

**功能边界**：
- in scope（MVP 候选）：4 种录入 + 语义检索 + 相关推荐 + 标签聚合 + 库内问答 + 自动整理（具体 MVP 边界阶段 4 收敛）
- out scope（后置）：浏览器插件一键剪藏、手机端、多端云同步、多人/多岗位共用、关系图谱可视化
- 不做：团队文档中心、内容共享平台、多租户

**资源约束**：开发交给 AI 编码助手（Claude Code），技术栈由 AI 按 local-first 最佳实践推荐；内容规模几千条量级；第一版电脑网页端。

**最大不确定性**：
- 贴链接抓取微信公众号正文的反爬可行性（需技术验证）
- 本地大模型在普通个人电脑上跑语义检索/RAG 的性能与效果

**最不该做的方向（防跑偏）**：不要做成又一个"团队文档协作平台/Wiki"——它是**个人的、本地的、AI 驱动的**，重心在"少录入、强找回、能关联"，不是多人编辑和审批流。

## 假设清单 v1
- A-001：微信公众号文章链接可被抓取正文（依据等级 C，需技术验证）
- A-002：普通个人电脑可跑动本地大模型完成语义检索/RAG，且效果可接受（依据等级 C，需技术验证）
- A-003：用户愿意"顺手"把内容丢进来——录入足够轻才会持续用（依据等级 B，产品成败关键）
- A-004：几千条规模下，本地向量检索性能足够（依据等级 B）
- A-005：第一版自用即可验证价值，多人版需求成立但暂不验证（依据等级 B）
