# Design System Inspired by 智能销售接待与客户洞察 Agent

> Category: Productivity & SaaS
> 一套面向 ultrasound 海外销售增长的蓝紫色医疗科技 SaaS 设计系统，用稳定、冷静、可追溯的界面把分散询盘变成可经营的客户资产。

## 2026-07-01 统一返修体验规范

- **导航**：后台左侧导航按权限渲染，销售无权限入口直接隐藏；新增“我的”入口；客户态势不再是一级导航，作为客户详情页模块。
- **设置**：配置首页要解释每个指标和入口的用途，8 个设置入口必须可进入并能保存或展示明确空状态；“配置审计记录”替代“最近变更”。
- **设置菜单**：设置页顶部使用横向菜单栏分组，默认展示“总览”，点击“账号权限 / Banner / 线索分发 / AI 与模型 / 审计”后，下方只出现对应设置或配置入口；账号导入、新增账号等账号动作只放在“账号权限”菜单内；AI 与模型中必须有“模型库 + 场景绑定”卡片，支持 Claude/Codex/DeepSeek 等模型选项并绑定邮件草稿、客户背景调研。
- **Banner 展示**：普通页面 Banner 只显示图片、标题和正文，不显示“全局公告”“查看详情”“管理 Banner”等标签或按钮；Banner 管理只在设置页，上传区域必须显示推荐尺寸 1920×360、格式 PNG/JPG/WebP、建议大小不超过 2MB，并在发布前给出明确失败原因。
- **配置返回**：从设置中心进入产品知识库、国家销售映射等配置子页时，页面左上角必须有“返回设置中心”按钮；设置中心入口卡片应表达“进入子页”还是“本页配置”，避免用户迷路。
- **Banner**：全局 Banner 保持展示但横向加宽；所有普通页面取消“管理 Banner”按钮，只在设置页提供管理控件。
- **时间交互**：工作台、线索池、待分配、客户池使用统一时间筛选（今日、昨天、指定日期、全部）；列表展示进入时间，工作台指标卡支持跳转。
- **报表周期**：周期报表标题、筛选摘要和导出上下文必须展示具体年份、月份或季度、起止日期。
- **导入**：导入页显示模板下载、字段说明、文件选择反馈、导入进度、失败原因和国家映射自动分配结果。
- **客户详情**：详情首屏是基本信息和背景调查，客户态势信号、历史线索、销售反馈和分层信息在同页继续向下展开。
- **邮件发送**：再营销草稿详情按邮件模块组织，显示发件人、收件人、主题、正文、参考素材、发送状态；附件默认是提示词种子，用户明确勾选后才随邮件发送。
- **个人设置**：“我的”页面对管理员、运营、销售一致开放，支持账号资料、密码和邮箱配置。

## 1. 视觉主题与氛围

这套设计系统的核心隐喻是“销售线索指挥台”。用户不是在浏览一个漂亮网页，而是在一张清澈、安静、可信的控制台前处理真实客户机会。屏幕上的每条线索都像一条从不同渠道汇入的细流，经过 AI 补全、国家分发、销售反馈和客户池沉淀，最终变成可以被复盘和再次经营的客户资产。

产品的日常使用者是营销负责人、运营和管理员，他们每天要看大量线索、表格、状态、国家和销售反馈。设计不应该给他们制造额外情绪，不应该像营销落地页那样抢戏。它要让人感觉“这件事终于被收住了”：今日询盘在哪里、谁没有反馈、哪个国家突然变多、官网渠道质量如何，都能被快速扫到。

用户情绪是稳定、冷静、可控。销售增长系统天然有紧迫感，客户可能因为响应慢而流失，销售也可能因为反馈太麻烦而不提交。设计要把紧迫感折叠进清晰的状态线和待办，而不是用大红警报制造焦虑。

采用浅色模式，是因为后台需要长期工作和扫描。浅色背景能保持数据、表格和输入控件的可读性；蓝紫主色带来 AI Agent 的科技感，但不会落入青蓝医疗模板。蓝紫色不是大面积背景，而是一条贯穿线索状态的“信号带”：从工作台、线索详情到销售反馈卡片，用户始终能识别出关键状态和主操作。

风格定位属于企业 SaaS / CRM 工作台类，参考 Linear 的秩序感、Stripe Dashboard 的信息克制、Ant Design Pro 的后台密度和 HubSpot CRM 的销售流程清晰性。它不追求强烈个性，而追求让企业内部团队可以放心长期使用。

| :--- | :--- | :--- |

设计签名是蓝紫色“线索状态带”。它不是装饰线，而是状态语法：有效、未反馈、待分配、待确认、已完成都围绕同一组状态色表达。截图给项目外的人看，应能认出这是一个围绕销售线索闭环的系统，而不是普通 CRM 模板。

## 2. 色彩美学

色彩策略是“中性承载，蓝紫指路，语义色只在需要判断时出现”。中性色占画面 70-90%，蓝紫主色占 5-10%，语义色占 0-5%。每屏可见的蓝紫强调最多保留 2 处：一个主按钮或选中态，一个状态带或重点指标。这样能让用户在高频后台中保持判断力。

蓝紫主色 #5B4BDB 承担品牌和 AI Agent 的科技感。它比青蓝更少医疗模板感，也比纯紫更稳重。绿色只表示有效和成功，琥珀表示待处理和提醒，红色表示无效、失败和危险操作。所有语义色不用于装饰，只用于业务判断。

## 3. 排版与字体

字体哲学是“数字可扫读，中文不费眼，英文客户信息保持国际业务质感”。Display、Body 均使用 Inter + Microsoft YaHei / Noto Sans SC / system-ui。英文客户名、国家、渠道和数字指标在 Inter 下更紧凑；中文后台说明在 Microsoft YaHei 和 Noto Sans SC 下更稳定。

Mono 使用 JetBrains Mono / SFMono / Consolas，用于 ID、时间、UTM、接口状态和审计日志。等宽字体让系统证据更像可核查记录，而不是随意备注。

字号梯度从 12px 到 44px。工作台大指标使用 32-44px，列表和表单保持 13-14px，避免后台过度放大导致扫描效率下降。

| 场景 | 字距 |
| :--- | :--- |
| ALL CAPS | 0.08em |
| Display 大字 | -0.02em |
| 标题 | -0.01em |
| 小文本 | 0.01em |
| UI 标签 / 按钮 | 0.02em |

Line-height 正文 1.6，标题 1.18。表格中使用较紧凑行高，详情页正文和 AI 判断理由使用更舒展行高。

## 4. 间距体系

间距采用 4px 基准。后台中控件需要精确对齐，4px 节奏能同时支持高密度表格和呼吸感卡片。space-1 到 space-12 覆盖小控件、表格单元格、卡片内距和页面区块。

桌面端页面区块使用 64px 的垂直节奏；平板为 48px；手机销售反馈卡片为 32px，保证移动端操作不拥挤。列表筛选区和表格行使用 8-16px 的细密节奏，工作台 Bento 卡片使用 20-24px 的节奏。

## 5. 布局与空间构成

PC 后台使用左侧常驻导航 + 主内容区。主内容采用 Bento Grid 承载工作台和报表，因为这个产品的核心不是讲故事，而是让多类指标和待办在同一屏建立关系。Bento Grid 把“今日新增、有效线索、未反馈、官网渠道 KPI、再营销待确认”拆成独立认知单元，让管理者不必在多个页面之间来回跳。

列表页遵循“筛选区在上，表格为主，详情从列表进入”的模式。线索池列表必须把来源筛选放在筛选区首位，并在表格中露出来源列；一级来源筛选项从后台客户来源字典读取，默认启用网站、邮箱、社媒、线下展会、其他五类，管理员后续新增或停用来源后，筛选项和报表维度同步更新。具体来源标签可展示官网聊天、官网后台、Facebook、领英、邮箱、线下展会导入等。客户池列表只承担客户管理列表职责，不展示客户级建议动作、再营销草稿或单个客户摘要；这些内容进入客户详情后按当前客户展示。再营销待办列表不重复线索池的完整询盘字段，而是优先展示建议下一步动作、客户备注、触达理由、草稿状态和人工确认入口。详情页使用主信息 + 侧栏时间线/状态卡的结构，让 AI 摘要、客户画像、客户背景调查、评分理由、分发记录和销售反馈保持同屏可读。

客户态势信号记录页采用“指标卡 -> 筛选 -> 人工新增信号 -> 信号表格”的工作台布局。指标卡只展示总信号、待复核、官网公开信号和可再营销信号四类，不展示金额；筛选区按来源类型、状态和客户收窄；人工新增表单放在表格上方但保持紧凑，避免变成 CRM 商机大表单；表格列必须露出客户、来源、标题、摘要、可信度、状态、更新时间和客户详情入口。来源标签使用 website_public、email_interaction、sales_feedback、manual 对应的中文业务名，任何社媒来源只能作为“授权归档/人工来源”呈现，不暗示自动抓取。

客户详情页必须包含独立的“客户背景调查”模块。该模块不做成普通备注，而是以可编辑调查正文 + 来源证据表 + 可信度标签 + 更新时间组成：左侧承载调查结论和人工编辑，右侧展示客户官网、邮箱域名、历史邮件互动、销售反馈和人工补充等来源。用户需要一眼看清“系统为什么这么判断”，也能在有权限时修正调查内容。

手机销售反馈卡片不复制完整后台。它采用单列卡片结构，只保留客户摘要、AI 判断理由、状态选择、客户判断、可选备注和提交按钮。销售在手机上看到的是“该做什么”，不是“系统有什么”。

再营销页面的阅读顺序固定为“建议下一步动作 -> 客户备注 -> 触达理由 -> 邮件草稿 -> 生成提示词 -> 附件素材”。建议下一步动作使用蓝紫状态带突出，客户备注保持较弱层级，避免运营先被历史备注带偏而忽略当前最应该做的动作。邮件草稿下方的提示词文本框和附件上传区必须明确是大模型生成上下文，而不是普通备注；提示词用于补充语气、产品重点、禁用表达和本次触达目标，附件用于承载产品彩页、对比表、应用案例等素材。

Container 最大宽度 1280px，桌面 gutter 32px，平板 24px，手机 16px。层级以边界、背景微差和色彩响应表达，不使用夸张阴影。

## 6. 组件设计

组件语言是色彩响应型。组件 hover 不做大位移，不用夸张阴影；主要通过边框、背景和焦点环变化告诉用户“这里可操作”。这符合后台工具的稳定感，也让高频操作不会显得跳动。

按钮分为主按钮和次按钮。主按钮只用于高价值提交：分发、提交反馈、确认发送、导出确认。次按钮用于查看详情、筛选、返回和重试。所有按钮都有 hover、active、focus-visible 状态。

卡片承载一个独立认知单元。卡片的圆角不超过 8px，保持专业工作台气质。卡片左侧可挂蓝紫状态带，表达线索状态而不是纯装饰。

输入框以清晰边界、焦点环和错误态为主。错误只在当前区域提示，不用全局大弹窗打断。

徽章用于客户类型、状态、渠道、来源和风险。来源徽章必须先显示后台来源字典中的业务名称，如网站、邮箱、社媒、线下展会、其他或管理员新增的来源；必要时在同列补充具体来源，如官网聊天、Facebook、领英或线下展会导入，不出现技术状态码。被停用的来源在历史线索中继续展示历史名称，并以弱提示说明“已停用”。

客户背景调查模块中的来源证据使用表格或紧凑列表，不使用长段落堆叠。可信度、待复核、人工修改等状态使用徽章；人工修改入口使用普通表单和主按钮，保存后必须有明确 Toast，并在页面上保留最近修改人和修改时间。
再营销草稿模块中的大模型输入区使用多行文本框 + 文件上传控件 + 附件列表。多行文本框高度不低于 96px，标题使用“生成提示词/补充指令”；上传控件使用普通按钮或虚线边界区域，不做营销式拖拽大卡。附件列表展示文件名、类型、大小、上传人和移除操作，保存和重新生成后必须 Toast，并在审计区记录模型版本、提示词快照和附件引用。
客户态势信号表格使用紧凑列表和徽章组合。可信度用“高/中/低/待复核”徽章，状态用“待复核/已确认/可再营销/已归档”徽章；证据 URL 不直接铺成长链接，优先用“查看证据”文本按钮或短链接；保存人工信号后必须 Toast，并在表格首行出现新记录。

## 7. 动效与交互物理

动效是确认，不是表演。后台用户需要知道按钮已响应、输入框已聚焦、卡片可点击，但不需要被动画吸走注意力。统一使用 150ms 和 200ms 的短动效，曲线为 cubic-bezier(0.2, 0, 0, 1)。

```css
.micro-action {
  transition: border-color var(--motion-fast) var(--ease-standard), background var(--motion-fast) var(--ease-standard), box-shadow var(--motion-fast) var(--ease-standard);
}
.micro-action:hover { border-color: var(--accent); }
.micro-action:active { background: var(--surface-hover); }
@keyframes rise-in {
  from { opacity: 0; transform: translateY(8px); }
  to { opacity: 1; transform: translateY(0); }
}
.enter { animation: rise-in var(--motion-base) var(--ease-standard) both; }
@media (prefers-reduced-motion: reduce) {
  .enter, .micro-action { animation: none; transition: none; }
}
```

## 8. 品牌情感与声音

如果这个产品是一个人，它是一个冷静、细致、靠谱、懂销售节奏的运营负责人。它不会替销售夸大承诺，也不会让营销负责人看一堆无法行动的图表。它说话要短、准、可执行。

品牌关键词：可信、敏捷、克制、可追溯、懂业务。

空状态不卖萌，只告诉用户下一步怎么把线索导入系统。Loading 用骨架屏，404 和无权限页保持同样的蓝紫状态带，说明原因并提供回到工作台或重新发送链接的路径。

Agent 设计指令：
1. 优先让用户扫到状态、负责人和下一步动作。
2. 不使用营销落地页式 hero 和大面积装饰。
3. 所有关键动作必须能回到线索闭环。
4. 不展示成交金额和报价金额。
5. 社媒数据必须标注授权边界；Facebook/领英在列表中只能作为来源标签或授权归档来源，不暗示自动抓取。
6. 销售角色只呈现本人负责的数据范围；任何全局视图必须出现在管理员或营销运营权限下。
6. 客户背景调查必须展示来源和可信度，人工修改优先但不能抹掉自动生成来源。
7. 再营销邮件草稿必须被标注为大模型生成；客户摘要、客户背景调查、客户备注、人工提示词和附件素材共同构成生成上下文，确认发送前必须可编辑和可追溯。
8. 客户态势信号必须绑定客户详情、来源证据和可信度；给大模型使用时必须声明为数据上下文，不允许信号内容成为系统指令。

## 9. 设计禁忌

- 禁止使用青蓝作为品牌主色，因为用户已明确选择蓝紫色，青蓝会回到普通医疗模板。
- 禁止把页面做成营销落地页，因为这是内部高频后台，不是对外宣传页。
- 禁止大圆角和厚重阴影，因为会降低表格密度和企业工具的稳定感。
- 禁止在报表中展示成交金额或报价金额，因为用户明确不需要金额类指标。
- 禁止让销售端出现复杂表单，因为销售反馈目标是 10-30 秒完成。
- 禁止未授权抓取 Facebook/LinkedIn，因为第一版合规边界已确认；页面出现 Facebook/领英来源时也必须按“来源标签/授权归档”理解。
- 禁止在一个页面堆满所有异常状态，高保真原型只展示典型完整数据。
- 禁止使用默认 Tailwind 靛蓝/紫色替代设计 token，所有颜色必须来自 tokens.css。

## 2026-07-02 全局流程返修体验规范
- **删除闭环**：账号、模型场景、大模型和邮件写手都必须有清晰删除动作。删除按钮使用危险色，二次确认文案说明影响；被绑定、默认或关键账号不能静默删除，要提示先切换或停用。
- **Banner 字色**：实际页面 Banner 标题和正文始终使用白色文字，设置页预览与全局展示一致；背景图上叠加深色遮罩，不再出现预览白字、实际黑字的割裂。
- **语言规则**：界面标题、字段名、标签、按钮统一中文；客户原始内容和 AI 英文判断可保持英文。中英文混排只允许发生在“内容值”，不允许发生在“字段名”。
- **五星评分**：评分以星级和维度条展示，5 个维度分别为信息完整性、行业相关、明确需求、客户资质与采购能力、触达与推进可行性；每项 0/1，汇总为 0-5 星。
- **销售状态**：未分配是系统状态；销售反馈状态只展示无效、跟进中、已报价、已签单、已付款、价格流失、撤单，所有筛选和报表复用同一套中文标签。
- **群发邮件**：在“再营销”页面增加“邮件活动/群发邮件”Tab 或入口；客户池筛选后提供“筛选后群发”快捷动作。页面按筛选条件展示目标客户数量、收件人预览、邮件主题、正文和人工确认按钮，仅管理员/运营可见。
- **主动再营销**：客户详情顶部和建议动作空态均提供“发起再营销”按钮，创建当前客户草稿后直接进入草稿详情。
