# 智能销售接待与客户洞察 Agent 开发版 PRD

## 2026-07-01 统一返修开发口径

- 设置管理：8 个入口必须可点击、可操作；设置首页要解释“销售账号、来源国家映射、全局 Banner”等指标含义，“最近变更”改为“配置审计记录”。
- Banner：全局展示区加宽；所有页面只展示 Banner 内容，不常驻“管理 Banner”按钮；Banner 管理只在设置页。
- 导入：CSV/XLSX 选择文件后必须显示文件状态、模板下载、字段要求、进度、失败原因和自动分配结果；导入按国家销售映射自动写入负责人。
- 权限：销售端无权限入口必须隐藏；后端 403 仅做兜底。销售可进入再营销，但仅能查看/发送自己分配或负责国家范围内客户。
- 邮件：新增“我的”模块配置个人邮箱；管理员/运营可配置主邮箱并给所有潜在客户发邮件，销售只能给自己范围内客户发邮件。再营销草稿详情显示发件人、收件人、主题、正文、参考素材和发送状态。
- 客户：左侧不再单列客户态势；客户态势并入客户详情。客户详情首屏补齐姓名、邮箱、单位、询盘需求、背景调查、进入时间和来源。
- 时间：工作台、线索池、待分配、客户池新增今日/昨天/具体日期/全部筛选和进入时间列；工作台指标卡必须可跳转到明细列表。周期报表必须显示年份、月份/季度、起止日期和筛选摘要。

## 0. 开发优先级

第一版只做询盘闭环 MVP：

1. 统一线索池
2. AI 接待与引导
3. 客户画像、客户背景调查和评分
4. 国家/区域分发
5. 销售反馈卡片
6. 未反馈提醒
7. 基础报表
8. 客户池基础分层
9. 设置管理：销售账号、角色权限、销售数据范围、全局顶部 Banner 和关键配置入口

## 1. 必须实现的功能模块

### FR-1 统一线索池

接收官网聊天、官网后台、邮箱、展会名片导入线索。每条线索保存来源渠道、接收时间、原始内容、客户联系方式、处理状态。支持疑似重复提示。

### FR-2 AI 接待与引导

客户咨询 ultrasound 时，AI 需要补齐身份、国家、应用、产品类型和具体需求。客户只说 Hi 时先接待；客户问价格时先问应用和产品类型，不直接承诺价格。

### FR-3 双客群画像

代理商画像字段：国家、历史业务、代理品牌、超声经验、业务规模。医生/终端用户画像字段：科室、临床应用、用过的机器、预算、目标产品和场景。

### FR-4 客户背景调查、背景摘要和评分

线索或客户导入后，系统自动结合客户官网公开信息、邮箱域名、历史邮件互动、销售反馈和人工补充生成客户背景调查，并在客户详情页展示。按身份明确、应用明确、需求明确三道硬门槛判断有效性。评分必须展示理由，销售或运营可人工修正；客户背景调查也必须支持人工编辑、版本记录和来源留痕。

### FR-5 国家/区域分发

按国家映射区域，再映射销售负责人。国家缺失时进入待补充。无负责人映射时进入待分配池。

### FR-6 销售反馈卡片

通过微信/邮件发送反馈链接。销售选择状态和客户判断，备注可选。状态包括已联系、已报价、无效、需跟进、已成交。

### FR-7 未反馈提醒

分发后 24 小时无反馈提醒销售，48 小时无反馈进入管理者关注。高意向线索优先提醒。

### FR-8 基础报表

报表统计询盘总量、有效数量、国家分布、渠道质量、代理商/医生占比、产品分类、销售反馈、未及时反馈、撤单、再营销数据、客户池变化。官网渠道需要支持 KPI 归因，统计官网询盘量、有效询盘数、有效率、AI 补全率、首次响应时效、分发时效、销售反馈率、未反馈数、进入客户池数量和进入再营销待办数量。不统计成交金额和报价金额。

### FR-9 客户池分层

客户池状态包括高意向客户、有效客户、已报价未回复、暂无明确需求、已转代理商跟进、无效客户、撤单/流失客户、已成交客户。

### FR-10 再营销草稿

大模型生成建议下一步动作、邮件草稿和触达理由，必须人工确认后发送。生成上下文包括客户摘要、客户背景调查、客户备注、历史邮件互动、销售反馈、人工补充提示词和上传附件素材。再营销页面需把建议下一步动作放在客户备注上方；草稿区下方必须提供提示词文本框和附件上传入口，支持运营补充写作目标、语气、产品重点、案例素材和禁用表达。第一版不自动群发。

### FR-11 客户态势信号记录

只记录轻量信号：官网公开信息、邮件互动、销售反馈中的机会、人工录入。不得做未授权社媒抓取。

### FR-12 管理配置

提供 Web 设置管理页，维护销售账号、管理员账号、角色权限、销售数据范围、全局顶部 Banner、国家/区域/销售映射、产品知识库、接待 playbook、渠道配置和提醒规则。销售角色默认只能查看分配给自己的询盘，不能看到其他销售负责的询盘；管理员和营销/运营按权限查看全部或指定范围。所有 Web 后台页面顶部展示统一 Banner；管理员可在设置管理页上传图片、编辑文案和跳转链接，并发布到全部页面。左侧“配置”默认进入设置管理首页；账号、权限、销售数据范围、Banner 和核心规则变更必须写入审计日志。

## 3. 状态流

```text
new_inquiry -> qualifying -> qualified -> assigned -> contacted -> quoted / follow_up_needed / invalid / won / lost
```

缺少国家或身份时：

```text
new_inquiry -> qualifying -> need_more_info -> qualifying
```

销售无反馈时：

```text
assigned -> no_feedback -> manager_attention
```

## 5. 技术建议

- 前端使用 React 或 Vue。
- 后端使用 Python 或 Node。
- 数据库使用 Postgres 或同等级关系型数据库。
- AI 部分先采用规则引擎 + LLM 结构化抽取和大模型草稿生成。
- 报表和提醒使用任务调度。
- 反馈链接需要过期时间和销售身份标记。
- 销售数据范围必须在后端接口层过滤，不能只靠前端隐藏菜单或按钮。

### 5.1 每步技术实现落点

| 步骤 | 推荐技术 | 技术放在哪里 |
|---|---|---|
| 线索进入统一线索池 | 官网聊天 Webhook/API、邮箱 IMAP 或 Microsoft Graph/Gmail API、官网后台 CSV/Excel 导入、展会名片 OCR/表格导入 | 渠道接入适配层，统一写入后端标准化服务 |
| AI 接待和引导提问 | LLM 对话编排、规则引擎、产品知识库检索、结构化字段抽取 | AI 接待服务和产品知识库，不能散落在前端页面硬编码 |
| 客户画像和背景调查 | LLM 结构化抽取、规则评分、客户官网公开信息、邮箱域名、历史邮件互动、销售反馈、人工修正记录 | 画像评分服务和背景调查服务，结果、来源、可信度和人工修改版本写入数据库 |
| 国家/区域分发 | Postgres 国家/区域/销售映射表、确定性路由规则、异常待分配队列 | 路由模块和设置管理后台 |
| 销售轻反馈 | 移动端 H5 反馈卡片、带过期时间的安全链接、邮件/微信消息中的反馈入口、SalesAssignment 负责人校验 | 反馈服务和权限校验层，销售只看到自己负责的最小反馈页 |
| 设置管理 | RBAC、账号状态、销售数据范围、全局 Banner、配置审计、设置总览、权限拦截 | Auth/Permission 服务和 Web 管理后台，统一控制菜单、按钮、接口权限、销售数据范围和页面顶部 Banner |
| 未反馈提醒和日报 | 定时任务、消息/邮件通知、报表聚合 SQL、缓存表或报表快照 | 后台任务服务和报表模块 |
| 客户池和再营销草稿 | 客户状态机、任务队列、邮件模板、大模型草稿生成、提示词输入、附件素材解析、建议下一步动作生成、人工审批流、退订/合规标记 | 客户池模块和再营销待办模块，发送前必须人工确认，客户备注上方展示 recommended_next_action，并把提示词、附件和模型版本写入 NurtureTask |

## 6. 不做

- 完整 CRM。
- 金额统计。
- Facebook/LinkedIn 未授权抓取。
- 自动群发再营销。
- 复杂销售业绩归因。

## 7. 开发顺序

1. 数据模型和统一线索池。
2. 设置管理：销售账号、权限、销售数据范围、全局 Banner、国家/销售映射、产品知识库。
3. AI 接待和字段抽取。
4. 客户画像、客户背景调查和评分。
5. 销售分发和反馈卡片。
6. 未反馈提醒。
7. 基础报表。
8. 客户池和再营销草稿。
