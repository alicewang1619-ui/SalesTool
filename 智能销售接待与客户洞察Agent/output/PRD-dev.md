# 智能销售接待与客户洞察 Agent 开发版 PRD

## 2026-07-01 统一返修开发口径

- 设置管理：顶部必须有菜单栏，按菜单切换展示对应设置；原 8 个入口必须可点击、可操作；账号导入/新增等账号动作只出现在账号管理菜单；设置首页要解释“销售账号、来源国家映射、全局 Banner”等指标含义；Banner 展示区不显示“全局公告/查看详情”等字样，Banner 管理需提供尺寸/格式/大小建议并修复上传发布失败；新增“大模型选择”配置，支持添加 Claude/Codex/DeepSeek 等模型并分别绑定 AI 接待、邮件草稿、客户背景调研；所有配置子页必须提供返回设置中心/上级菜单；“最近变更”改为“配置审计记录”。
- Banner：全局展示区加宽；所有页面只展示 Banner 内容，不常驻“管理 Banner”按钮；Banner 管理只在设置页。
- 导入：CSV/XLSX 选择文件后必须显示文件状态、模板下载、字段要求、进度、失败原因和自动分配结果；导入按国家销售映射自动写入负责人。
- 权限：销售端无权限入口必须隐藏；后端 403 仅做兜底。销售可进入再营销，但仅能查看/发送自己分配或负责国家范围内客户。
- 邮件：新增“我的”模块配置个人邮箱；管理员/运营可配置主邮箱并给所有潜在客户发邮件，销售只能给自己范围内客户发邮件。再营销草稿详情显示发件人、收件人、主题、正文、参考素材和发送状态。
- 客户：左侧不再单列客户态势；客户态势并入客户详情。客户详情首屏补齐姓名、邮箱、单位、询盘需求、背景调查、进入时间和来源。
- 时间：工作台、线索池、待分配、客户池新增今日/昨天/具体日期/全部筛选和进入时间列；工作台指标卡必须可跳转到明细列表。周期报表必须显示年份、月份/季度、起止日期和筛选摘要。

## 0. 开发优先级

第一版只做询盘闭环 MVP：

1. 统一线索池
2. AI 接待与引导
3. 客户画像、客户背景调查和评分
4. 国家/区域分发
5. 销售反馈卡片
6. 未反馈提醒
7. 基础报表
8. 客户池基础分层
9. 设置管理：销售账号、角色权限、销售数据范围、全局顶部 Banner 和关键配置入口

## 1. 必须实现的功能模块

### FR-1 统一线索池

接收官网聊天、官网后台、邮箱、展会名片导入线索。每条线索保存来源渠道、接收时间、原始内容、客户联系方式、处理状态。支持疑似重复提示。

### FR-2 AI 接待与引导

客户咨询 ultrasound 时，AI 需要补齐身份、国家、应用、产品类型和具体需求。客户只说 Hi 时先接待；客户问价格时先问应用和产品类型，不直接承诺价格。

### FR-3 双客群画像

代理商画像字段：国家、历史业务、代理品牌、超声经验、业务规模。医生/终端用户画像字段：科室、临床应用、用过的机器、预算、目标产品和场景。

### FR-4 客户背景调查、背景摘要和评分

线索或客户导入后，系统自动结合客户官网公开信息、邮箱域名、历史邮件互动、销售反馈和人工补充生成客户背景调查，并在客户详情页展示。按身份明确、应用明确、需求明确三道硬门槛判断有效性。评分必须展示理由，销售或运营可人工修正；客户背景调查也必须支持人工编辑、版本记录和来源留痕。

### FR-5 国家/区域分发

按国家映射区域，再映射销售负责人。国家缺失时进入待补充。无负责人映射时进入待分配池。

### FR-6 销售反馈卡片

通过微信/邮件发送反馈链接。销售选择状态和客户判断，备注可选。状态包括已联系、已报价、无效、需跟进、已成交。

### FR-7 未反馈提醒

分发后 24 小时无反馈提醒销售，48 小时无反馈进入管理者关注。高意向线索优先提醒。

### FR-8 基础报表

报表统计询盘总量、有效数量、国家分布、渠道质量、代理商/医生占比、产品分类、销售反馈、未及时反馈、撤单、再营销数据、客户池变化。官网渠道需要支持 KPI 归因，统计官网询盘量、有效询盘数、有效率、AI 补全率、首次响应时效、分发时效、销售反馈率、未反馈数、进入客户池数量和进入再营销待办数量。不统计成交金额和报价金额。

### FR-9 客户池分层

客户池状态包括高意向客户、有效客户、已报价未回复、暂无明确需求、已转代理商跟进、无效客户、撤单/流失客户、已成交客户。

### FR-10 再营销草稿

大模型生成建议下一步动作、邮件草稿和触达理由，必须人工确认后发送。生成上下文包括客户摘要、客户背景调查、客户备注、历史邮件互动、销售反馈、人工补充提示词和上传附件素材。再营销页面需把建议下一步动作放在客户备注上方；草稿区下方必须提供提示词文本框和附件上传入口，支持运营补充写作目标、语气、产品重点、案例素材和禁用表达。第一版不自动群发。

### FR-11 客户态势信号记录

只记录轻量信号：官网公开信息、邮件互动、销售反馈中的机会、人工录入。不得做未授权社媒抓取。

### FR-12 管理配置

提供 Web 设置管理页，维护销售账号、管理员账号、角色权限、销售数据范围、全局顶部 Banner、国家/区域/销售映射、产品知识库、接待 playbook、大模型选择、渠道配置和提醒规则。设置页顶部使用菜单栏分组，点击菜单后只展示对应配置域，避免一屏堆叠全部配置；账号导入、账号新增和账号编辑只放在账号权限菜单，不在所有菜单右上角常驻。销售角色默认只能查看分配给自己的询盘，不能看到其他销售负责的询盘；管理员和营销/运营按权限查看全部或指定范围。所有 Web 后台页面顶部展示统一 Banner，但普通展示区只显示图片、标题、正文，不显示“全局公告/查看详情”等字样；管理员可在设置管理页上传图片、查看推荐尺寸/格式/大小、编辑文案并发布到全部页面。大模型设置需返回模型库、场景绑定和当前生效配置，支持新增 Claude/Codex/DeepSeek 等模型选项，并分别绑定 AI 接待、邮件草稿、客户背景调研。左侧“配置”默认进入设置管理首页；账号、权限、销售数据范围、Banner、大模型选择和核心规则变更必须写入审计日志。所有配置子页需有返回设置中心或返回上级菜单按钮。

## 5. 技术建议

- 前端使用 React 或 Vue。
- 后端使用 Python 或 Node。
- 数据库使用 Postgres 或同等级关系型数据库。
- AI 部分先采用规则引擎 + LLM 结构化抽取和大模型草稿生成。
- 报表和提醒使用任务调度。
- 反馈链接需要过期时间和销售身份标记。
- 销售数据范围必须在后端接口层过滤，不能只靠前端隐藏菜单或按钮。

## 2026-07-01 开发补充：AI 场景与邮件写手

- `AIModelConfig` 需要支持 `options`、`use_cases`、`use_case_bindings`、`email_writers`、`default_email_writer`，其中 `use_cases` 和 `email_writers` 均允许管理员新增与编辑。
- 默认场景至少包含 `customer_research`（客户背景调查）和 `email_draft`（邮件草稿写作）；前端设置页必须提供“添加场景”入口。
- 邮件写手角色默认初始化 Doraemon、Mario、Pikachu、Totoro、Baymax、Nemo，每个角色保存英文标识、中文名、风格、技能数组、适用场景和启用状态。
- 再营销任务增加 `writer_role_key`，草稿详情页可选择邮件写手角色；保存和重新生成都要持久化该选择，并把角色风格/技能写入 `prompt_context_snapshot`。
- 设置页账号权限菜单移除顶部重复入口卡片，仅保留账号表、权限矩阵、导入账号和新增账号等真实操作。

- 客户信息导入必须提供可下载 CSV 模板、明确支持 CSV/XLSX、展示必填字段/建议字段/示例行，选择文件后显示文件名、校验进度、成功/失败明细和自动分配结果。

## 2026-07-01 三次统一返修开发口径
- 邮件接口配置表单必须显示字段标题：发信邮箱、发件人名称、SMTP Host、SMTP Port、SMTP 用户名、SMTP 密码/应用专用密码、测试收件人邮箱；TLS 和全局主邮箱开关必须有清晰标签。
- AI 与模型配置必须以“场景优先”为主流程：先选择或新增模型使用场景，再为该场景绑定大模型；场景默认包含客户背景调查、邮件草稿写作，并允许后续新增报价跟进、线索摘要等场景。
- 大模型库必须支持新增真实可启用配置：模型 Key、模型名称、供应商、API Base URL、Endpoint Path、鉴权方式、API Key/Token（前端输入，后端只回显已配置状态）、能力说明、适用场景和启用状态。
- AI 与模型页面不得一次性铺满模型库和邮件写手；模型库、场景绑定、邮件写手均采用下拉/折叠式摘要展示，点击后再展开或打开弹窗编辑。
- 邮件写手角色配置必须通过角色摘要卡/下拉选择进入编辑弹窗；弹窗自上而下编辑英文标识、中文名、风格、技能、适用场景、启用状态，不能使用横向表格行内编辑。
- 登录页必须是独立视觉页面，未登录状态不展示后台左侧菜单、后台 Banner 或后台业务导航；登录成功后才进入后台 AppShell。
- 后台左侧菜单必须固定在视口，不跟随主内容滚动；主内容区域独立滚动，所有详情页顶部提供清晰返回按钮（返回列表/返回上一页）。
- 客户来源字典必须改成下拉/折叠展示：默认展示来源类型摘要和数量，点击某一类型后仅展示该类型下来源项，可新增、编辑、启停和保存，避免所有来源一次性铺开。
