# 智能销售接待与客户洞察 Agent 开发版 PRD

## 0. 开发优先级

第一版只做询盘闭环 MVP：

1. 统一线索池
2. AI 接待与引导
3. 客户画像和评分
4. 国家/区域分发
5. 销售反馈卡片
6. 未反馈提醒
7. 基础报表
8. 客户池基础分层

## 1. 必须实现的功能模块

### FR-1 统一线索池

接收官网聊天、官网后台、邮箱、展会名片导入线索。每条线索保存来源渠道、接收时间、原始内容、客户联系方式、处理状态。支持疑似重复提示。

### FR-2 AI 接待与引导

客户咨询 ultrasound 时，AI 需要补齐身份、国家、应用、产品类型和具体需求。客户只说 Hi 时先接待；客户问价格时先问应用和产品类型，不直接承诺价格。

### FR-3 双客群画像

代理商画像字段：国家、历史业务、代理品牌、超声经验、业务规模。医生/终端用户画像字段：科室、临床应用、用过的机器、预算、目标产品和场景。

### FR-4 背景摘要和评分

按身份明确、应用明确、需求明确三道硬门槛判断有效性。评分必须展示理由，销售可修正。

### FR-5 国家/区域分发

按国家映射区域，再映射销售负责人。国家缺失时进入待补充。无负责人映射时进入待分配池。

### FR-6 销售反馈卡片

通过微信/邮件发送反馈链接。销售选择状态和客户判断，备注可选。状态包括已联系、已报价、无效、需跟进、已成交。

### FR-7 未反馈提醒

分发后 24 小时无反馈提醒销售，48 小时无反馈进入管理者关注。高意向线索优先提醒。

### FR-8 基础报表

报表统计询盘总量、有效数量、国家分布、渠道质量、代理商/医生占比、产品分类、销售反馈、未及时反馈、撤单、再营销数据、客户池变化。不统计成交金额和报价金额。

### FR-9 客户池分层

客户池状态包括高意向客户、有效客户、已报价未回复、暂无明确需求、已转代理商跟进、无效客户、撤单/流失客户、已成交客户。

### FR-10 再营销草稿

AI 生成草稿和触达理由，必须人工确认后发送。第一版不自动群发。

### FR-11 客户态势信号记录

只记录轻量信号：官网公开信息、邮件互动、销售反馈中的机会、人工录入。不得做未授权社媒抓取。

### FR-12 管理配置

维护国家/区域/销售映射、产品知识库、接待 playbook、提醒规则和权限。

## 2. 核心数据对象

| 对象 | 关键字段 |
|---|---|
| Inquiry | inquiry_id、source_channel、received_at、raw_content |
| Conversation | conversation_id、speaker_role、message_summary |
| CustomerProfile | customer_id、customer_type、country、application_area、product_interest、background_summary |
| LeadScore | score_level、score_reason |
| SalesAssignment | sales_owner、region |
| SalesFeedback | status、customer_judgement、note |
| CustomerPoolRecord | pool_status |
| NurtureTask | draft_content、approval_status |
| CustomerSignal | signal_source、signal_summary |

## 3. 状态流

```text
new_inquiry -> qualifying -> qualified -> assigned -> contacted -> quoted / follow_up_needed / invalid / won / lost
```

缺少国家或身份时：

```text
new_inquiry -> qualifying -> need_more_info -> qualifying
```

销售无反馈时：

```text
assigned -> no_feedback -> manager_attention
```

## 4. 验收标准

| AC ID | 验收点 |
|---|---|
| AC-1 | 官网线索进入统一线索池 |
| AC-2 | 邮箱线索导入后生成线索 |
| AC-3 | 客户只说 Hi 时 AI 能开启接待 |
| AC-4 | 客户要买 ultrasound 时 AI 能追问身份、国家、应用和产品类型 |
| AC-5 | 代理商客户展示代理商画像字段 |
| AC-6 | 医生客户展示医生画像字段 |
| AC-7 | 身份、应用、需求明确时可判定有效 |
| AC-8 | 国家明确后自动匹配销售 |
| AC-9 | 国家缺失时进入待补充或待分配 |
| AC-10 | 销售能提交状态和客户判断 |
| AC-11 | 24 小时无反馈提醒，48 小时进入管理者关注 |
| AC-12 | 日报展示核心指标 |
| AC-13 | 销售反馈后客户进入客户池 |
| AC-14 | 再营销草稿必须人工确认 |
| AC-15 | 未授权社媒数据不被抓取 |

## 5. 技术建议

- 前端使用 React 或 Vue。
- 后端使用 Python 或 Node。
- 数据库使用 Postgres 或同等级关系型数据库。
- AI 部分先采用规则引擎 + LLM 结构化抽取。
- 报表和提醒使用任务调度。
- 反馈链接需要过期时间和销售身份标记。

### 5.1 每步技术实现落点

| 步骤 | 推荐技术 | 技术放在哪里 |
|---|---|---|
| 线索进入统一线索池 | 官网聊天 Webhook/API、邮箱 IMAP 或 Microsoft Graph/Gmail API、官网后台 CSV/Excel 导入、展会名片 OCR/表格导入 | 渠道接入适配层，统一写入后端标准化服务 |
| AI 接待和引导提问 | LLM 对话编排、规则引擎、产品知识库检索、结构化字段抽取 | AI 接待服务和产品知识库，不能散落在前端页面硬编码 |
| 客户画像和背景补全 | LLM 结构化抽取、规则评分、公开/授权数据源查询、人工修正记录 | 画像评分服务，结果和来源写入数据库 |
| 国家/区域分发 | Postgres 国家/区域/销售映射表、确定性路由规则、异常待分配队列 | 路由模块和管理员配置后台 |
| 销售轻反馈 | 移动端 H5 反馈卡片、带过期时间的安全链接、邮件/微信消息中的反馈入口 | 反馈服务和权限校验层，销售只看到最小反馈页 |
| 未反馈提醒和日报 | 定时任务、消息/邮件通知、报表聚合 SQL、缓存表或报表快照 | 后台任务服务和报表模块 |
| 客户池和再营销草稿 | 客户状态机、任务队列、邮件模板、人工审批流、退订/合规标记 | 客户池模块和再营销待办模块，发送前必须人工确认 |

## 6. 不做

- 完整 CRM。
- 金额统计。
- Facebook/LinkedIn 未授权抓取。
- 自动群发再营销。
- 复杂销售业绩归因。

## 7. 开发顺序

1. 数据模型和统一线索池。
2. 管理配置：国家/销售映射、产品知识库。
3. AI 接待和字段抽取。
4. 客户画像和评分。
5. 销售分发和反馈卡片。
6. 未反馈提醒。
7. 基础报表。
8. 客户池和再营销草稿。
