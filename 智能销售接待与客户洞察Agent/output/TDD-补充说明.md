# TDD 补充说明

## 本次需求

再营销邮件草稿需要由大模型结合客户摘要、客户背景调查、客户备注、销售反馈、人工提示词和上传附件素材生成。草稿详情页必须在邮件草稿下方提供“生成提示词/补充指令”文本框和附件上传入口，并支持重新生成、人工确认发送和留痕。

## 待并入正式 TDD 的验收点

| 编号 | 测试点 | 期望结果 |
| :--- | :--- | :--- |
| TDD-NURTURE-01 | 草稿详情页展示大模型输入区 | 邮件草稿下方显示提示词文本框、附件上传入口、附件列表和重新生成按钮 |
| TDD-NURTURE-02 | 保存提示词 | 提示词写入 `NurtureTask.generation_prompt`，并记录修改人和修改时间 |
| TDD-NURTURE-03 | 上传附件素材 | 合规附件写入 `attachment_refs`，不合规类型提示失败且不影响已编辑草稿 |
| TDD-NURTURE-04 | 重新生成草稿 | 系统使用客户摘要、背景调查、客户备注、销售反馈、提示词和附件解析结果作为上下文重新生成草稿 |
| TDD-NURTURE-05 | 发送前合规校验 | 草稿仍需人工确认；客户退订、不再联系、提示词或附件包含禁用承诺时禁止发送 |
| TDD-NURTURE-06 | 列表提示词/附件状态 | 再营销待办列表展示提示词状态和附件数量，帮助运营优先处理缺少素材的草稿 |
 
## 2026-07-02 本批返修验收补充

| 编号 | 测试点 | 期望结果 |
| :--- | :--- | :--- |
| TDD-FIX-01 | 客户详情评分缺失 | mock 客户无 `score_summary` 时页面仍渲染，显示“待评分”和 0/5 星，不抛出 `max_score` 异常。 |
| TDD-FIX-02 | 工作台明细移除 | 访问工作台首页不出现客户明细表、不出现行级“查看详情”按钮，仍能通过指标/快捷入口跳转列表页。 |
| TDD-FIX-03 | 线索池发邮件 | 在线索池点击某行“发邮件”后调用再营销任务接口并跳转任务详情，任务的 `customer_id` 等于该线索对应客户。 |
| TDD-FIX-04 | 宽松导入 | 使用中英文表头 CSV/XLSX 均能创建导入任务；缺失非核心字段时记录待确认原因但不整批失败。 |
| TDD-FIX-05 | 配置总览去重 | 配置中心总览只有顶部 Tab 导航，不再渲染“配置菜单总览”重复卡片。 |
| TDD-FIX-06 | 大模型启停 | 停用某模型后无法保存为默认模型或场景绑定；运行时选择模型会回退到启用模型。 |
| TDD-FIX-07 | 自定义知识库板块 | 保存产品/竞品/市场或自定义板块知识后，列表筛选和 AI 上下文均能按板块区分。 |
