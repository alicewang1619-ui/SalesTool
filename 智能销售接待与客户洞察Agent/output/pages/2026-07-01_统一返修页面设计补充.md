# 2026-07-01 统一返修页面设计补充

## 适用页面

本补充文档覆盖以下页面，并优先级高于同目录旧页面文档里的过时描述：

| 页面 | 返修设计要求 |
|---|---|
| 工作台首页 | 新增总询盘入口；支持今日、昨天、指定日期、全部筛选；今日询盘、总询盘、有效线索、未反馈指标卡点击后跳转到带筛选参数的线索/客户/待分配列表。 |
| 线索池列表 | 展示线索进入时间；增加时间筛选；保留来源筛选；从工作台跳转进入时显示筛选摘要。 |
| 渠道导入 | 提供 CSV/XLSX 模板下载、字段说明、文件选择反馈、导入进度、失败原因和按国家销售映射自动分配结果。 |
| 待分配列表 | 展示进入时间和待分配原因；支持今日、昨天、指定日期、全部筛选；仅显示需要人工确认的缺失映射、停用销售或冲突记录。 |
| 客户池列表 | 展示首个询盘进入时间和最近更新时间；支持国家、产品、状态和时间筛选。 |
| 客户详情 | 首屏优先展示姓名、邮箱、单位、国家、产品、来源、进入时间、询盘需求/内容和客户背景调查；下方展示客户分层、负责人、历史线索、销售反馈、客户态势信号。 |
| 再营销待办列表 | 管理员/运营看全量；销售可进入但只看本人分配或负责国家范围内客户。 |
| 草稿详情确认发送 | 以邮件发送模块组织页面：发件人邮箱、收件人邮箱、主题、正文、参考素材、发送状态、人工确认发送；附件默认作为 AI 写邮件参考种子，不默认作为邮件附件。 |
| 周期报表视图 | 标题、筛选摘要、指标卡和导出路径明确显示年份、月份/季度、起止日期。 |
| 设置管理 | 顶部增加设置菜单栏，按菜单切换对应设置域；原 8 个设置入口全部可用；账号导入/新增只放入账号权限菜单；指标卡增加解释；新增大模型选择配置并支持新增 Claude/Codex/DeepSeek 等模型及场景绑定；“最近变更”改为“配置审计记录”；Banner 管理只在此处。 |
| 我的账号与邮箱 | 管理员、运营、销售统一可进入；支持账号资料、密码、个人邮箱配置；销售邮箱用于本人客户的营销邮件发送。 |
| 全局 Banner | 普通页面只显示 Banner 图片、标题和正文，不显示“全局公告”“查看详情”“管理 Banner”等标签或按钮；展示宽度加大，避免页面显窄；设置页 Banner 上传需显示推荐尺寸、格式和大小并给出清晰失败原因。 |
| 权限导航 | 前端按角色隐藏无权限入口；销售端不再出现点进去才提示无权限的菜单项；后端仍保持 403 兜底。 |

## 页面联动

1. 工作台指标卡生成带查询参数的跳转，例如 `/admin/leads?time_scope=today`、`/admin/leads?score=valid`、`/admin/assignments/pending?time_scope=today`。
2. 客户池点击“查看详情”进入客户详情；客户态势信号作为详情页内模块，不再从左侧导航进入。
3. 导入成功后立即展示自动分配统计：已自动分配、待人工确认、失败行；待人工确认可跳转待分配列表。
4. 再营销邮件发送前必须同时展示发件人与收件人；未配置邮箱时按钮禁用并引导到“我的”或设置页主邮箱配置。

|---|---|---|---|

| 我的账号与邮箱 | 可编辑本人账号、密码、邮箱 | 可编辑本人账号、密码、邮箱 | 可编辑本人账号、密码、邮箱 |
| 再营销待办 | 全量 | 全量 | 仅本人客户范围 |
| 再营销发送邮件 | 全部潜在客户 | 全部潜在客户 | 仅本人客户范围 |
| 客户详情态势 | 全量可看可新增 | 全量可看可新增 | 本人客户可看，不可新增全局信号 |
| 报表 | 可见 | 可见 | 隐藏入口 |
| 待分配 | 可见 | 可见 | 隐藏入口 |

## 2026-07-01 实现回写

- `AppShell` 已按角色过滤导航：销售仅显示工作台、线索池、客户池、再营销、我的；配置、报表、待分配和客户态势入口不再出现，后端 403 仍作为兜底。
- `GlobalBanner` 已移除普通页面“管理 Banner”按钮；Banner 管理仅保留在配置中心，展示区加宽并只承担公告展示。
- `LeadImportPage` 已提供模板下载、字段说明、选中文件名、导入进度、失败行、自动分配数和待人工确认数。
- `DashboardPage`、`LeadsPage`、`PendingAssignmentsPage`、`CustomersPage` 已补齐今日、昨天、指定日期、全部历史时间线和筛选摘要；工作台指标卡可跳转对应列表。
- `CustomerDetailPage` 首屏展示姓名、邮箱、单位、询盘需求、来源摘要、首次询盘时间和背景调查；客户态势信号作为详情模块呈现。
- `NurtureTasksPage` 与 `NurtureTaskDetailPage` 已按邮件系统组织：显示发件人、收件人、主题、正文、参考附件、邮件状态和人工确认发送。
- `SettingsPage` 已将 8 个设置入口改为真实入口/设置区；销售账号支持新增、编辑和软启停；审计记录解释其回溯价值。
- 配置子页返回交互追加要求：从设置中心进入产品知识库、国家销售映射等子页时，页面头部必须提供“返回设置中心”，避免进入子配置后无法回到配置中心。
- `MyProfilePage` 已新增，管理员、运营、销售都可修改自己的显示名称、密码和个人发件邮箱配置。
