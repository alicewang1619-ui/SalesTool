# 周期报表视图

| 字段 | 内容 |
| :--- | :--- |
| 所属产品 | 智能销售接待与客户洞察 Agent |

## 2. 页面概述

### 2.1 一句话定位
按日/月/季/年查看询盘量、有效量、国家、渠道、产品和销售反馈。

### 2.2 存在理由
1. 没有该页面，线索闭环中的对应动作会回到人工微信、邮件或表格。
2. 没有统一入口，管理者无法判断线索状态、销售反馈和客户池变化。
3. 该页面把 PRD 中“少填、快看、可追溯”的要求落为具体操作。

### 2.3 页面目标（可衡量）

| 目标维度 | 具体指标 |
| :--- | :--- |
| 效率 | 关键操作在 3 次点击内完成 |
| 准确性 | 状态、负责人、来源和时间字段完整展示 |
| 可追溯 | 人工修改、导出、发送或提交均能回到记录 |

### 2.4 不做什么
- 不展示成交金额和报价金额，因为 PRD 明确排除金额类指标。
- 不做完整 CRM 的复杂商机阶段管理，第一版聚焦线索闭环。
- 不做未授权 Facebook/LinkedIn 抓取。

## 3. 用户与场景

### 3.1 目标用户

| 维度 | 描述 |
| :--- | :--- |
| 角色 | 管理员、营销/运营、销售，按权限看到不同动作 |
| 技术熟练度 | 中等，熟悉表格和基础后台操作 |
| 使用设备 | PC 浏览器为主 |
| 使用姿势 | 高频扫描、快速判断、少量确认 |

### 3.2 核心使用场景

**场景 A：正常处理**
- 时机：用户进入 周期报表视图 执行日常任务。
- 前置：已登录或持有有效安全链接。
- 动作：查看关键信息，完成筛选、确认、反馈、导入或查看详情。
- 情绪：清楚、稳定、知道下一步。

**场景 B：异常兜底**
- 时机：数据缺失、链接过期、权限不足或任务失败。
- 前置：当前流程无法自动完成。
- 动作：查看原因，进入待处理、重试、返回工作台或请求管理员处理。
- 情绪：不被卡住，知道谁来处理。

### 3.3 用户核心诉求
1. 快速看到状态和下一步。
2. 不重复录入系统已有信息。
3. 能追溯 AI 判断、销售反馈和人工修改来源。

## 4. 信息架构

### 4.1 区域划分
1. 顶部标题与关键动作。
2. 核心指标或摘要区。
3. 主操作区。
4. 明细表格、时间线或表单区。
5. 异常提示和权限说明区。

### 4.2 区域内容
- 标题区展示页面名称、阶段、当前状态和主操作。
- 主体区展示与 报表中心 相关的核心业务数据。
- 明细区展示客户、线索、反馈、报表或配置记录。

## 5. 界面内容

### 5.1 元素清单
- 页面标题：周期报表视图。
- 状态标签：有效、未反馈、待分配、待确认、已完成等。
- 筛选或表单控件：来源、国家、客户类型、产品、销售、周期。
- 主操作按钮：查看详情、确认、保存、导入、导出或提交反馈。
- 记录列表：至少展示客户名称、国家、渠道、产品、负责人、更新时间。

### 5.2 辅助信息
- AI 判断理由、数据来源、更新时间、导出脱敏说明或链接有效期说明。

## 6. 交互流程

### 6.1 页面加载与初始化
- 校验登录态或安全链接。
- 拉取页面所需数据。
- 按业务时间倒序展示列表或记录。

### 6.2 核心操作流程

```text
进入周期报表视图
  -> 查看关键状态和上下文
  -> 执行主操作
  -> 系统保存结果并记录来源
  -> 跳转或回写到下游页面：指标明细、导出确认弹窗
```

### 6.3 辅助操作
- 搜索、筛选、排序、重试、返回上级和导出确认。

## 7. 状态与边界

### 7.1 页面状态全景

| 状态 | 触发条件 | 页面表现 |
| :--- | :--- | :--- |
| 初始态 | 首次进入 | 展示标题、骨架或默认数据 |
| 加载态 | 请求中 | 骨架屏或按钮 loading |
| 完整态 | 数据可用 | 展示 mock 数据和关键动作 |
| 空态 | 无数据 | 给出导入、配置或返回入口 |
| 失败态 | 请求失败 | 展示原因和重试 |
| 无权限 | 角色不允许 | 隐藏或置灰动作并说明原因 |

### 7.2 边界与容错

| 边界场景 | 处理策略 |
| :--- | :--- |
| 网络异常 | 保留当前输入，允许重试 |
| 会话过期 | 跳回登录页或提示重新发送链接 |
| 连续错误 | 显示可理解原因，不吞掉错误 |
| 异常输入 | 当前字段就地提示 |
| 并发操作 | 以最新保存记录为准并保留操作人 |
| 权限不足 | 动作置灰或进入无权限页 |

## 8. 业务规则与校验

### 8.1 字段定义

| 字段 | 说明 |
| :--- | :--- |
| source_channel | 官网聊天、官网后台、邮箱、展会名片等来源 |
| country | 决定区域和销售负责人 |
| customer_type | 代理商、Doctor、Hospital、Clinic 等 |
| product_interest | Trolley、Portable、Handheld 或具体型号 |
| feedback_status | 已联系、已报价、无效、需跟进、已成交 |

### 8.2 校验规则

| 校验项 | 时机 | 规则 | 失败反馈 |
| :--- | :--- | :--- | :--- |
| 必填项 | 提交时 | 关键字段不能为空 | 标记待补充 |
| 权限 | 操作前 | 角色必须允许动作 | 动作不可用 |
| 链接有效期 | 打开反馈卡片 | 7 天内有效 | 提示重新发送 |
| 导出字段 | 导出前 | 不包含成交金额和报价金额 | 阻止导出 |

### 8.3 特殊业务规则
- 销售判断优先于 AI 评分，但 AI 判断理由保留。
- 再营销必须人工确认后发送。
- 官网 KPI 只统计渠道归因和时效，不统计金额。

## 9. 页面流转

### 9.1 流转图

```text
工作台 / 左侧导航
  -> 周期报表视图
  -> 指标明细、导出确认弹窗
```

### 9.2 与其他模块的关系
- 与线索池、客户池、报表中心和系统配置保持统一状态。
- 涉及销售反馈时同步写入 SalesFeedback 和 ReportMetric。

### 9.3 扩展预案

| 扩展方向 | 触发条件 | 改造点 |
| :--- | :--- | :--- |
| CRM 集成 | 阶段三启动 | 增加 external_id 和同步状态 |
| 企业微信通知 | 销售反馈入口需要统一 | 增加 notification provider |
| 深度客户态势 | 授权数据源明确 | 增加信号采集和来源审计 |

## 10. 数据需求

### 10.1 页面需要什么数据

| 数据 | 来源 | 用途 |
| :--- | :--- | :--- |
| 当前用户与角色 | Auth 模块 | 权限控制 |
| 线索/客户/反馈记录 | 后端业务模块 | 展示主体内容 |
| 配置字典 | 系统配置 | 渲染国家、产品、销售选项 |
| 报表聚合 | ReportMetric | 展示指标 |

### 10.2 页面提交什么数据

| 数据 | 触发时机 | 用途 |
| :--- | :--- | :--- |
| 筛选条件 | 查询时 | 获取列表 |
| 状态或配置修改 | 保存时 | 更新业务记录 |
| 反馈或确认结果 | 提交时 | 写入闭环 |

### 10.3 明确排除的数据
- 成交金额、报价金额。
- 未授权社媒私域数据。
- 与当前页面无关的完整 CRM 字段。

## 11. 验收标准

### 11.1 功能验收

| 编号 | 验收点 | 通过标准 |
| :--- | :--- | :--- |
| F-01 | 页面可打开 | 有权限用户能进入页面 |
| F-02 | 主数据展示 | 关键字段完整可读 |
| F-03 | 主操作可完成 | 操作后有明确反馈 |
| F-04 | 跳转可用 | 能进入下游页面 |
| F-05 | 权限控制 | 无权限动作不可执行 |

### 11.2 体验验收

| 编号 | 验收点 | 通过标准 |
| :--- | :--- | :--- |
| U-01 | 扫描效率 | 3 秒内看懂核心状态 |
| U-02 | 文案清晰 | 无技术黑话和占位符 |
| U-03 | 移动适配 | H5 页面点击区域足够 |

| 编号 | 验收点 | 通过标准 |
| :--- | :--- | :--- |
| S-01 | 登录态 | 未登录不可访问后台 |
| S-02 | 敏感信息 | 按角色展示和导出脱敏 |
| S-03 | 反馈链接 | 过期或非负责人不可提交 |
