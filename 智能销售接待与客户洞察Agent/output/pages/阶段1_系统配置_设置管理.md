# 设置管理

## 1. 文档元信息
| 字段 | 内容 |
| :--- | :--- |
| 所属产品 | 智能销售接待与客户洞察 Agent |
| 所属模块 | 系统配置 |
| 适用端 | Web |
| 页面路由 | `/admin/settings` |
| 页面类型 | 管理后台总览 / 设置工作台 |
| 所属阶段 | 阶段1(MVP) |

## 2. 页面概述

### 2.1 一句话定位
为管理员提供完整的系统设置入口，用于维护销售账号、角色权限、全局顶部 Banner、国家/区域销售映射、产品知识库、渠道导入参数、AI 接待边界、大模型选择和提醒规则。页面顶部必须有设置菜单栏，按菜单切换展示对应配置，避免一屏堆叠全部内容。

### 2.2 存在理由
1. 当前导航中已有“配置”，但缺少承接它的管理员总页面，用户进入后无法集中管理销售账号和权限。
2. 国家分发、销售反馈、客户池沉淀和报表统计都依赖管理员配置，必须有可视化入口保证配置完整。
3. 分散配置页需要一个总览页展示异常、最近变更和配置健康度，避免销售离职、区域缺负责人、权限误配后无人发现。

### 2.3 页面目标
| 目标维度 | 具体指标 |
| :--- | :--- |
| 可发现 | 左侧“配置”默认进入设置管理首页 |
| 可管理 | 管理员可在 3 次点击内进入销售账号、角色权限、全局 Banner、国家映射、产品知识库和大模型选择 |
| 可控 | 账号、权限和关键规则修改必须记录操作人、时间和修改前后值 |
| 可预警 | 展示停用销售仍有负责区域、国家无负责人、权限异常、渠道断连等配置风险 |

### 2.4 不做什么
- 不做完整 CRM 的组织架构、绩效和佣金管理。
- 不展示成交金额、报价金额和销售业绩金额。
- 不允许普通销售通过该页面修改自己负责区域或系统级权限。

## 3. 用户与场景

### 3.1 目标用户
| 角色 | 可见内容 | 可操作内容 |
| :--- | :--- | :--- |
| 超级管理员 | 全部设置、全部审计记录 | 新增/停用账号、分配角色、修改权限、维护所有规则 |
| 营销/运营负责人 | 线索、渠道、产品知识库、提醒规则 | 维护渠道导入、产品知识库、提醒策略；不可授予超级管理员 |
| 销售主管 | 销售账号、负责区域、未反馈规则 | 查看和申请调整销售负责区域；不可修改 RBAC |
| 销售 | 仅个人账号和反馈入口状态 | 修改个人基础信息，不可进入系统级设置 |

### 3.2 核心场景
**场景 A：新增销售账号**
- 管理员进入设置管理，点击“新增销售”。
- 填写姓名、邮箱、手机号、区域、默认语言和角色。
- 系统校验邮箱唯一性，保存后自动进入国家/区域销售映射候选负责人。

**场景 B：销售离职或暂停负责区域**
- 管理员将销售状态改为停用。
- 系统检查其负责国家/区域和未完成线索。
- 若仍有负责区域，要求先转移负责人或将相关线索进入待分配池。

**场景 C：调整权限**
- 管理员在角色权限矩阵中勾选功能权限。
- 系统提示影响范围，例如“销售将无法导出客户邮箱”。
- 保存后记录审计日志，并立即影响菜单、按钮和接口权限。

**场景 D：检查配置健康度**
- 管理员查看页面顶部风险卡片。
- 发现“6 个国家无销售负责人”“2 个渠道同步失败”。
- 点击风险项进入对应配置页处理。

## 4. 信息架构

### 4.1 页面区域
1. 顶部标题与主操作：新增销售、导入账号、查看审计。
2. 配置健康度：账号状态、权限风险、国家映射完整率、渠道同步状态。
3. 顶部 Banner 管理：上传图片、编辑文案、配置跳转、预览和发布。
4. 顶部菜单栏：总览、账号权限、Banner、线索分发、AI 与模型、审计。
5. 设置分组入口：账号与权限、销售分发、产品与 AI、渠道与提醒。
6. 销售账号管理：销售列表、角色、状态、负责区域、最近反馈。
7. 角色权限矩阵：角色与功能权限的勾选和只读说明。
8. 最近变更：展示操作人、对象、变更内容和时间。

### 4.2 设置分组
| 分组 | 子页面/能力 | 说明 |
| :--- | :--- | :--- |
| 账号与权限 | 销售账号、管理员账号、角色权限、登录安全 | 决定谁能进入系统、看到什么、执行什么 |
| 全局展示 | 顶部 Banner、公告图、跳转链接、展示状态 | 决定所有后台页面顶部统一展示内容 |
| 销售分发 | 国家区域销售映射、待分配规则、负责人转移 | 决定线索自动分发结果 |
| 产品与 AI | 产品知识库、AI 接待 Playbook、评分规则、大模型选择 | 决定 AI 引导、摘要、评分和再营销草稿生成依据 |
| 渠道与提醒 | 渠道导入、Webhook、邮箱同步、未反馈提醒 | 决定线索进入和闭环提醒 |

## 5. 界面内容

### 5.1 关键元素
| 元素 | 类型 | 默认态 | 操作后 | 禁用条件 |
| :--- | :--- | :--- | :--- | :--- |
| 新增销售 | 按钮 | 可点击 | 打开销售账号表单 | 非管理员 |
| 导入账号 | 按钮 | 可点击 | 上传账号模板 | 非管理员 |
| Banner 上传 | 文件控件 | 当前 Banner | 选择图片后生成预览 | 无 Banner 管理权限 |
| Banner 发布 | 按钮 | 草稿或当前版本 | 发布到全部后台页面 | 未上传图片或无权限 |
| Banner 下线 | 按钮 | 已启用 | 所有页面隐藏 Banner 或切换为默认公告 | 无权限 |
| 大模型选择 | 下拉框/卡片 | 当前默认模型 | 选择并保存默认模型 | 模型不可用或无权限 |
| 配置健康卡 | 指标卡 | 展示风险数 | 点击进入对应配置页 | 无 |
| 销售账号表 | 表格 | 按更新时间倒序 | 可编辑角色、状态、负责区域 | 无账号管理权限 |
| 角色权限矩阵 | 表格/勾选 | 展示当前权限 | 保存后立即生效 | 非超级管理员 |
| 设置入口卡 | 导航卡 | 展示完成度和风险 | 跳转到子配置页 | 无权限时只读或隐藏 |
| 审计日志 | 时间线 | 展示最近 20 条 | 可筛选对象和操作人 | 无审计权限 |

### 5.2 字段要求
| 字段 | 说明 | 校验 |
| :--- | :--- | :--- |
| user_id | 用户唯一标识 | 系统生成，不可编辑 |
| name | 销售或管理员姓名 | 必填 |
| email | 登录邮箱和通知邮箱 | 必填、唯一、邮箱格式 |
| phone / wechat | 手机或微信通知标识 | 至少配置一种通知方式 |
| role | 超级管理员、营销/运营、销售主管、销售 | 必填 |
| account_status | active、inactive、locked | 必填 |
| owner_regions | 负责区域/国家 | 销售角色至少配置一个或允许为空进入待分配候选 |
| permission_set | 权限集合 | 只能由超级管理员修改 |
| banner_image_url | 顶部 Banner 图片地址 | 支持 PNG、JPG、WebP；推荐 1920 x 360 |
| banner_title / banner_subtitle | Banner 标题和说明 | 标题必填，说明可选 |
| banner_target_url | Banner 跳转地址 | 必须为系统内页面或允许的 HTTPS 链接 |
| banner_status | draft、active、inactive | 发布时必须写入审计日志 |

## 6. 交互流程

### 6.1 默认进入流程
```text
左侧导航点击“配置”
  -> 打开设置管理首页
  -> 查看配置健康度
  -> 点击账号权限 / 国家映射 / 产品知识库 / 渠道配置
  -> 完成配置修改
  -> 写入审计日志并回到设置管理查看状态
```

### 6.2 Banner 上传发布流程
```text
打开设置管理
  -> 进入顶部 Banner 管理
  -> 上传图片并编辑标题、说明和跳转链接
  -> 系统校验图片格式、大小、链接安全性和权限
  -> 生成预览
  -> 发布到全部后台页面
  -> 记录操作人、时间和版本
```

### 6.3 新增销售流程
```text
点击新增销售
  -> 填写姓名、邮箱、通知方式、角色和负责区域
  -> 系统校验邮箱唯一、角色权限、区域冲突
  -> 保存账号
  -> 同步到销售候选人和国家区域映射
  -> 发送初始登录通知
```

### 6.4 权限修改流程
```text
打开角色权限矩阵
  -> 修改功能权限
  -> 系统展示影响范围
  -> 二次确认
  -> 保存权限集
  -> 菜单、按钮、接口权限同步生效
  -> 写入审计日志
```

## 7. 状态与边界

| 状态 | 触发条件 | 页面表现 |
| :--- | :--- | :--- |
| 正常 | 配置完整 | 健康卡展示绿色状态 |
| 有风险 | 有国家无负责人、销售停用但仍有关联线索 | 风险卡置顶，支持跳转处理 |
| 无权限 | 当前角色无设置权限 | 隐藏敏感数据，跳转无权限页 |
| 冲突 | 两名管理员同时编辑同一配置 | 保存时提示配置已更新，要求刷新后重试 |
| 空账号 | 尚未创建销售账号 | 展示新增销售和导入模板入口 |
| 渠道异常 | Webhook/邮箱同步失败 | 在渠道卡展示最近失败时间和重试入口 |
| Banner 缺失 | 未上传或已下线 | 所有页面展示默认系统公告 Banner |
| Banner 图片过大 | 超过限制 | 阻止上传并提示压缩 |
| Banner 链接不安全 | 链接不在白名单 | 阻止发布 |

## 8. 业务规则与校验

1. 超级管理员不能删除自己，只能由另一名超级管理员停用。
2. 停用销售前必须处理其负责国家/区域和未完成线索。
3. 销售角色默认只能查看和反馈分配给自己的线索；不能导出全量客户联系方式。
4. 营销/运营可以查看报表和维护内容配置，但不能授予超级管理员权限。
5. 国家/区域映射保存后不自动重分发历史线索，需要管理员手动确认。
6. 权限修改、账号停用、负责人转移、导出配置都必须写入审计日志。
7. 账号导入必须返回成功、失败和重复账号明细。
8. 全局 Banner 发布后必须同步到所有 Web 后台页面顶部，移动端自动折行，不遮挡页面主标题和操作按钮。
9. Banner 上传只允许 PNG、JPG、WebP，建议比例为 16:3；外链跳转必须符合安全白名单。

## 9. 页面流转

```text
设置管理

  -> 国家区域销售映射
  -> 产品知识库
  -> 渠道配置

```

## 10. 数据需求

### 10.1 页面读取
| 数据 | 来源 | 用途 |
| :--- | :--- | :--- |
| 当前用户与角色 | Auth 模块 | 控制设置入口和操作权限 |
| 用户账号列表 | User / StaffAccount | 展示销售和管理员账号 |
| 角色权限集合 | Role / Permission | 渲染权限矩阵 |
| 全局 Banner 配置 | BannerConfig | 渲染所有页面顶部 Banner |
| 国家区域销售映射 | TerritoryMapping | 展示配置完整率和风险 |
| 渠道连接状态 | ChannelConfig | 展示同步状态和异常 |
| 配置审计日志 | AuditLog | 追溯变更 |

### 10.2 页面提交
| 数据 | 触发时机 | 用途 |
| :--- | :--- | :--- |
| 账号新增/编辑 | 保存账号时 | 创建或更新用户账号 |
| 角色权限变更 | 保存权限矩阵时 | 更新 RBAC |
| 负责人转移 | 转移销售负责区域时 | 更新映射并影响后续分发 |
| 渠道/提醒入口配置 | 点击子页面保存时 | 更新系统配置 |
| Banner 图片、文案和链接 | 发布 Banner 时 | 更新全部后台页面顶部展示 |

## 12. 生产化落地状态（06-30-2026）
- 真实页面路由：`/admin/settings`，作为左侧“配置”默认入口，展示统一 Banner、设置管理标题、导入账号/审计日志/新增销售按钮、四张健康度卡、顶部 Banner 管理、配置风险、设置入口、销售账号表、角色权限矩阵和最近变更。
- 返修要求：`/admin/settings` 顶部展示设置菜单栏，默认“总览”；点击不同菜单后，只展示对应设置域，不再一次性堆叠全部设置。新增“AI 与模型”菜单，展示大模型选择、产品知识库、AI 接待 Playbook 和评分规则入口。
- 真实后端接口：`GET /api/settings/overview` 聚合 summary、当前全局 Banner、设置入口、销售/管理员账号、权限矩阵、配置风险和最近审计日志；仅管理员/运营可访问，销售角色返回 403。
- 真实后端接口补充：`GET /api/settings/overview` 返回 `ai_model` 配置；`PUT /api/settings/ai-model` 保存默认大模型选择并写入配置审计。
- 前端兼容要求：若本地开发服务尚未重启导致 `overview.ai_model` 暂缺，AI 与模型菜单必须使用默认大模型配置兜底展示，避免设置页因旧响应崩溃；保存动作仍以真实接口结果为准。
- 销售账号创建：`POST /api/settings/sales-users` 保存 name、email、password、role、data_scope、enabled，密码使用后端 hash，邮箱重复返回结构化 409，成功后写入 `settings_sales_user_created` 审计日志。
- Banner 发布：`PUT /api/settings/banner` 校验 PNG/JPG/WebP data URL、`/assets/` 或 HTTPS 图片来源，并校验跳转链接必须是 `/admin` 内部路径或 HTTPS；发布后停用旧 Banner，新增 active Banner，`GET /api/banner` 立即读取最新版本。
- 权限矩阵：`PUT /api/settings/permissions` 将角色权限保存到 `SystemSetting`，设置首页重新读取时展示最新 permissions，并写入 `settings_permissions_updated` 审计日志。
- 前端支持上传 Banner 图片到 data URL 预览、编辑 Banner 标题/说明/跳转链接、发布到全部页面、新增销售账号、保存权限矩阵和读取最近变更；页面操作全部通过真实 API，不再使用静态 mock。
- 浏览器验收：Playwright 打开 `/admin/settings` 可见设置管理、顶部 Banner 管理、设置入口、销售账号表和角色权限矩阵；新增测试销售账号成功并显示；发布 `Browser Banner Notice` 后全局 Banner 和预览同步更新；保存权限矩阵后显示成功提示并进入最近变更；控制台 warning/error 为 0。
