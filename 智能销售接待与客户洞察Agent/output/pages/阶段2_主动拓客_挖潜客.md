# 主动拓客 · 挖潜客

- 路由：`/admin/prospecting`
- 权限：管理员、运营可访问；销售不可见、不可直接访问。
- 目标：把“主动寻找潜在客户”从线索池导入中拆出来，形成方案生成、候选分析、人工确认入库的闭环。

## 页面结构

1. 顶部标题：`挖潜客`，说明为“根据产品、区域和平台式客户画像生成拓客方案，候选客户确认后进入线索池。”
2. 拓客方案表单：品牌（选填）、产品侧重点、目标区域、结构化客户画像、来源渠道标签多选（Google、LinkedIn、Google Maps、Facebook、人工来源及自定义来源）。
3. 客户画像配置：行业/机构类型、岗位或采购角色、公司规模或渠道类型、应用场景、意图关键词、排除条件，交互上参考 Facebook / LinkedIn Audience 的标签式配置。
4. AI 拓客方案卡：展示本次拓客策略、渠道节奏和注意事项。
5. 候选客户表：展示客户名称、国家、类型、产品、来源渠道、来源说明、来源链接、AI 匹配理由、评分、状态和操作。

## 关键交互

- 点击“生成拓客方案”后调用后端创建 ProspectingPlan，并返回候选客户。
- 系统必须使用结构化画像字段生成搜索 query 与候选客户，不允许只按品牌/国家拼通用候选；品牌为空时仍可按产品、区域和客户画像生成。
- 来源渠道支持选择预设项并直接输入新增来源，提交后后端必须保留自定义来源并用于候选客户来源链接。
- 候选客户的“来源链接”必须可点击打开新窗口，便于用户核验客户从哪里挖来。
- 点击“确认入库”后创建正式线索，候选状态改为“已入库”，线索保留来源渠道、来源说明和来源链接。
- 点击“丢弃”后候选状态改为“已丢弃”，不进入线索池。

## 边界

- LinkedIn/Facebook 不做未授权抓取，只作为公开搜索或授权归档来源。
- 候选客户未确认前不得出现在正式线索池。
- 重复确认不得创建重复线索。
