# 智能销售接待与客户洞察 Agent 方案 PPT 大纲

## p01 首页

标题：智能销售接待与客户洞察 Agent

副标题：把每月 300+ 条 ultrasound 询盘，从分散消息变成可跟进、可反馈、可分析、可再营销的客户资产。

## p02 执行摘要

这不是一个普通客服机器人，也不是一个重 CRM。第一版要做的是询盘闭环 MVP：统一线索池、AI 接待、客户画像评分、国家分发、销售轻反馈和基础报表。

## p03 核心痛点场景

询盘来自官网、邮箱、官网后台、展会名片等多个入口。人工沟通、查背景、整理、分发平均约 10 分钟/条；销售反馈 50%+ 缺失，导致管理者看不清渠道质量和销售跟进情况。

## p04 为什么这问题值 200 万

价值不是“替人回复消息”，而是修复销售增长漏斗：每月 300+ 条询盘、约 50 小时人工处理、20%-30% 客户损失、50%+ 销售反馈黑洞。

## p05 产品设计：询盘闭环

多渠道询盘进入统一线索池，AI 补齐身份、国家、应用、产品类型、需求，再按国家/区域分发销售，销售用轻量卡片反馈，结果进入报表和客户池。

## p06 产品设计：双客群画像

代理商关注重点国家、历史业务、代理品牌、超声经验和业务规模。医生/终端用户关注科室、临床应用、用过的机器、预算、目标产品和场景。

## p07 产品设计：反馈与报表

销售反馈只需要状态 + 客户判断 + 可选备注。管理者看非金额型报表：询盘总量、有效数量、国家分布、渠道质量、客户类型占比、产品分类、销售反馈、未及时反馈、客户池变化。

## p08 技术方案

模块包括渠道接入、AI 接待、画像评分、国家路由、反馈卡片、报表、客户池、再营销待办。数据治理要求来源留痕、权限控制、UTC 内部时间和合规边界。

## p09 每步技术实现落点

线索进入用渠道接入适配层；AI 接待用 LLM 编排、规则引擎和产品知识库；画像评分用结构化抽取、规则评分和来源留痕；国家分发用 Postgres 映射表和确定性路由；销售反馈用移动端 H5 卡片和安全链接；日报提醒用定时任务和报表聚合；客户池和再营销用状态机、任务队列和人工审批流。

## p10 阶段路线图

阶段一 MVP 跑通询盘闭环；阶段二增强再营销和客户态势信号；阶段三连接 CRM、企业微信、商业数据源和更复杂的态势监督。

## p11 本次训练营要做的阶段

本次落地做阶段一 MVP：官网聊天/官网后台/邮箱/展会名片进入统一线索池，AI 接待引导，画像评分，国家分发，销售轻反馈，基础报表。
