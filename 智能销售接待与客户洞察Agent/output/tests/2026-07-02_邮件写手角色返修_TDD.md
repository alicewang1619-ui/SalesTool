# 2026-07-02 邮件写手角色返修 TDD

## 设置管理
- SETTINGS-WRITER-ROLE-02：邮件写手配置弹窗不展示中文名字段；管理员可编辑英文角色名、能力与技能方向、角色目标、相关技能、背景定义、角色定义标签、适用场景和启用状态。
- SETTINGS-WRITER-ROLE-03：默认写手包含 `ReplyMirror`，且 `/api/ai/email-writers` 返回的角色名和前端下拉均为英文显示。
- SETTINGS-WRITER-ROLE-04：保存修改后的角色定义后，再营销重新生成会读取最新角色定义进入 Prompt，上下文快照包含完整角色字段。

## 再营销详情
- NURTURE-WRITER-PURPOSE-01：写邮件页面展示本次邮件目的，保存和重新生成时写入 `email_purpose`。
- NURTURE-WRITER-PROMPT-01：后端生成上下文必须包含角色目标、能力方向、技能、背景定义和标签，且正文不能输出角色说明文。
- NURTURE-WRITER-CHANGE-01：切换角色或目的后必须重新调用生成接口，返回的草稿和上下文快照跟随最新角色/目的变化。
- NURTURE-WRITER-PROMPT-02：每个写手必须有可配置的英文后台执行提示词 `prompt_directive`；重新生成上下文必须包含该字段，并把它作为模型写作指令的一部分。
- NURTURE-WRITER-CHANGE-02：同一客户、同一人工提示词下切换不同写手，生成的英文正文必须在语气、段落结构、CTA 或承接角度上有可见差异，不能只是角色名变化。
## UI 追加验收
- SETTINGS-WRITER-ROLE-05：邮件写手 Select 只显示英文角色名，详细说明在 Tooltip/Popover 中查看。
- SETTINGS-WRITER-ROLE-06：编辑、删除当前写手只出现在选中角色详情卡片底部，不再出现在邮件写手卡片顶部操作区。
- NURTURE-WRITER-UI-01：再营销写邮件页的写手 Select 下方不得重复出现选中角色名按钮或 hover-only 说明框；应显示固定 DT 地贴提示写手配置进入后台 Prompt。
