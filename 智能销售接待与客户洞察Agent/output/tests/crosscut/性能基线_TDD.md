# TDD 测试设计文档：性能基线

## 1. 单元元信息
| 字段 | 内容 |
| :--- | :--- |
| 测试层 | 性能基线 |
| 单元短码 | PERF |
| 对应上游产物 | 技术方案.md 性能与任务队列；PRD 非功能指标 |
| 覆盖范围 | 首屏、列表查询、导入、异步任务、AI 生成、报表聚合 |

## 2. 测试策略
- P1：MVP 月询盘 300+ 场景下，常用页面和接口保持流畅。
- P2：批量导入、AI 任务和报表聚合不阻塞主请求。

## 3. 测试用例清单
| ID | 用例名称 | 优先级 | Given | When | Then | 预期失败原因 | 来源 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| P-PERF-01 | 后台首屏在目标时间内可交互 | P1 | 普通办公网络和 300 条线索数据 | 打开工作台 | 首屏关键内容 3 秒内可交互 | 未实现时首屏加载过慢影响运营 | 技术方案 MVP规模 |
| P-PERF-02 | 线索池列表查询不超过目标时间 | P1 | 1000 条线索、五类来源筛选 | 查询线索池 | 95 分位响应 <= 800ms | 未实现时筛选体验卡顿 | 技术方案 查询 |
| P-PERF-03 | 客户详情背景调查异步加载不阻塞详情 | P1 | 背景调查任务处理中 | 打开客户详情 | 详情先展示基础信息，调查区显示处理中 | 未实现时客户详情被 AI 任务拖慢 | 技术方案 Celery |
| P-PERF-04 | 导入任务进入队列不阻塞请求 | P2 | 上传 500 行 Excel | 提交导入 | API 快速返回任务 ID，后台处理 | 未实现时请求超时 | 技术方案 任务队列 |
| P-PERF-05 | 再营销重新生成不阻塞页面 | P1 | 提示词和附件已保存 | 点击重新生成 | 页面显示生成中，任务异步执行 | 未实现时按钮长时间卡死 | PRD AC-20 |
| P-PERF-06 | 报表聚合读取快照 | P2 | 多来源、多日期数据 | 打开报表中心 | 使用 ReportMetric 快照，不实时全表扫 | 未实现时报表随数据增长变慢 | 技术方案 报表 |
| P-PERF-07 | Banner 图片发布后页面加载不劣化 | P1 | 发布符合限制的 Banner 图 | 打开多个后台页面 | Banner 不导致首屏明显超时 | 未实现时图片过大影响全站 | PRD AC-17 |
| P-PERF-08 | 附件解析任务有大小和超时保护 | P2 | 上传接近限制的 PDF | 解析附件素材 | 超时可失败并提示，不拖垮 worker | 未实现时附件解析阻塞队列 | 技术方案 附件解析 |

## 4. Mock 策略
| 依赖 | Mock 方式 | 理由 | 关联用例 |
| :--- | :--- | :--- | :--- |
| AI/附件解析 | 可控延迟 fake worker | 验证异步和超时 | P-PERF-03、P-PERF-05、P-PERF-08 |
| 数据库 | 种子数据集 300/1000/5000 | 验证 MVP 和增长边界 | P-PERF-02、P-PERF-06 |

## 5. 实现顺序建议
P1：P-PERF-01 -> P-PERF-02 -> P-PERF-03 -> P-PERF-05 -> P-PERF-07。  
P2：P-PERF-04 -> P-PERF-06 -> P-PERF-08。
