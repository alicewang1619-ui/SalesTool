# TDD 测试设计文档：MVP 业务闭环

## 1. 单元元信息
| 字段 | 内容 |
| :--- | :--- |
| 测试层 | E2E 业务流 |
| 单元短码 | FLOW |
| 对应上游产物 | 页面清单.md；设计决策蓝图.md；PRD详细版.md AC-1~AC-20 |
| 所属产品/模块 | 从线索进入到客户沉淀和再营销确认的端到端链路 |

## 2. 测试策略
- P0：线索进入、AI/人工补全、分发、销售反馈、客户详情和权限隔离必须跑通。
- P1：再营销提示词/附件、来源字典配置、Banner 配置和报表入口必须跑通。
- 不测：真实外部系统连通性，使用测试替身。

## 4. Mock 策略
| 依赖 | Mock 方式 | 理由 | 关联用例 |
| :--- | :--- | :--- | :--- |
| 官网聊天/邮箱 | 测试夹具或 mock endpoint | 保持 E2E 稳定 | E-FLOW-01、E-FLOW-02 |
