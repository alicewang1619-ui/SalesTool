## 用例
| ID | 优先级 | Given | When | Then | 预期失败原因 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| PAGE-REPORT-HOME-01 | P0 | 管理员进入报表中心 | 打开页面 | 展示日报、月报、季报、年报入口和非金额核心指标，顶部 Banner 与后台一致 | 缺入口会让报表路径断裂 |
| PAGE-REPORT-HOME-02 | P0 | 报表 API 有真实聚合数据 | 加载指标 | 页面数字与后端聚合一致，且不展示成交金额/报价金额 | 展示被 PRD 排除的金额指标会偏离范围 |
| PAGE-REPORT-HOME-03 | P1 | 点击周期报表入口 | 跳转页面 | 携带周期参数打开周期报表视图 | 假按钮无法进入分析链路 |
| PAGE-REPORT-HOME-04 | P1 | 报表数据生成中 | 打开页面 | 显示生成状态、更新时间和重试入口 | 长任务状态缺失会造成误判 |
| PAGE-REPORT-HOME-05 | P2 | 大数据量指标聚合 | 请求报表 | 查询有时间范围、分页/聚合限制，响应满足性能基线 | 全量扫描会影响生产可用性 |


## 生产化 TDD 映射（06-30-2026）

- `backend/tests/test_api_contract.py` 新增 `report_home` 专项契约测试，覆盖未登录拦截、销售角色禁止访问、管理员非金额指标/周期入口/渠道质量/官网 KPI 聚合，以及重新生成接口排队与审计。
- 后端绿灯标准：`py -m pytest .\tests -q -k report_home` 必须全部通过；全量后端 `py -m pytest .\tests -q` 必须保持通过。
- 前端绿灯标准：`npm.cmd run build` 必须通过；浏览器验收打开 `/admin/reports` 后必须看到全局 Banner、报表中心首页、日报/月报/季报/年报入口、今日询盘/有效线索/未反馈/官网 KPI、渠道质量、官网归因率和重新生成入口。
- 页面不得出现“成交金额”“报价金额”等金额类字段；该项作为报表首页范围控制的固定回归断言。
