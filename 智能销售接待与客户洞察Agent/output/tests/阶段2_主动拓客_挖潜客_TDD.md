# 主动拓客 · 挖潜客 TDD

## 覆盖目标

- 管理员/运营能按结构化客户画像生成拓客方案和候选潜客。
- 画像字段必须覆盖行业/机构类型、岗位或采购角色、公司规模或渠道类型、应用场景、意图关键词和排除条件。
- 候选潜客必须带来源渠道、来源说明和来源链接。
- 候选确认后进入线索池，并保留来源追溯。
- 销售角色不可访问挖潜客页面和接口。

## 用例

| ID | 优先级 | 场景 | Given | When | Then |
|---|---|---|---|---|---|
| P-API-01 | P0 | 生成画像驱动拓客方案 | 管理员已登录，填写结构化客户画像 | 提交选填品牌、区域、行业、角色、场景、关键词、渠道和本次挖取数量 | 返回拓客方案、渠道节奏和指定数量的候选潜客 |
| P-API-07 | P0 | 品牌选填和自定义来源 | 管理员已登录，品牌为空且新增自定义来源渠道 | 提交产品、区域、画像字段、预设来源和自定义来源 | 计划创建成功，自定义来源不会被后端丢弃，并出现在候选客户来源中 |
| P-API-08 | P0 | 挖取数量上限 | 管理员已登录，填写结构化客户画像 | 提交 `candidate_limit=7` | 返回 7 条候选潜客；提交 0 或 101 时返回校验错误 |
| P-API-02 | P0 | 搜索链接包含画像关键词 | 管理员已生成候选 | 查看候选来源链接 | `source_url` 包含目标区域、行业/机构类型、角色或意图关键词 |
| P-API-03 | P0 | 来源追溯 | 管理员已生成候选 | 查看候选列表 | 每条候选包含 `source_channel`、`source_note`、`source_url` |
| P-API-04 | P0 | 确认入库 | 候选状态为待确认 | 点击确认入库 | 创建正式线索，线索保留来源链接和来源说明 |
| P-SEC-01 | P0 | 销售越权 | 销售账号已登录 | 访问 `/api/prospecting` | 返回 403 |
| P-EDGE-01 | P1 | 重复确认 | 候选已入库 | 再次确认 | 不创建重复线索，返回已入库状态 |

## 红线

- 不允许把 LinkedIn/Facebook 做成未授权抓取。
- 不允许只按品牌/国家生成泛泛候选，必须使用客户画像字段；品牌为空时不能阻断拓客方案生成。
- 来源渠道不能只接受固定枚举，用户新增的自定义来源必须参与计划、候选客户和入库追溯。
- 每次挖取必须由 `candidate_limit` 限定数量，后端不能因为渠道多或重复提交而无限生成候选。
- 不允许候选未确认就进入线索池。
- 不允许候选入库丢失来源链接。
