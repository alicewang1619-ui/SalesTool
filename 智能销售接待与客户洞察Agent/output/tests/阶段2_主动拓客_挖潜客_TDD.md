# 主动拓客 · 挖潜客 TDD

## 覆盖目标

- 管理员/运营能生成拓客方案和候选潜客。
- 候选潜客必须带来源渠道、来源说明和来源链接。
- 候选确认后进入线索池，并保留来源追溯。
- 销售角色不可访问挖潜客页面和接口。

## 用例

| ID | 优先级 | 场景 | Given | When | Then |
|---|---|---|---|---|---|
| P-API-01 | P0 | 生成拓客方案 | 管理员已登录 | 提交品牌、区域、画像和渠道 | 返回拓客方案、渠道节奏和至少一条候选潜客 |
| P-API-02 | P0 | 来源追溯 | 管理员已生成候选 | 查看候选列表 | 每条候选包含 `source_channel`、`source_note`、`source_url` |
| P-API-03 | P0 | 确认入库 | 候选状态为待确认 | 点击确认入库 | 创建正式线索，线索保留来源链接和来源说明 |
| P-SEC-01 | P0 | 销售越权 | 销售账号已登录 | 访问 `/api/prospecting` | 返回 403 |
| P-EDGE-01 | P1 | 重复确认 | 候选已入库 | 再次确认 | 不创建重复线索，返回已入库状态 |

## 红线

- 不允许把 LinkedIn/Facebook 做成未授权抓取。
- 不允许候选未确认就进入线索池。
- 不允许候选入库丢失来源链接。
