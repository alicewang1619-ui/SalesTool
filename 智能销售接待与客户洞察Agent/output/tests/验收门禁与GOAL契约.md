# 验收门禁与 GOAL 契约：智能销售接待与客户洞察 Agent

> 给 Coding Agent：这是本项目开发终止条件。先读本文件，再读 `测试索引.md` 和各层 `_TDD.md`。按 P0 到 P1 顺序逐条红绿重构，全部 P0 + P1 转绿前不得宣称交付。

## 0. /goal 生产化硬门禁
- 本项目开发阶段按 `/goal 全流程一键编排` 原则执行：交付目标是面向真实用户的生产应用，不是原型或 demo。
- 所有功能必须建立在真实后端 API 与持久化数据之上，前端请求、后端处理、数据库读写必须真实打通；只做前端、硬编码数据、抄原型 mock 数据、空实现或假按钮均视为未完成。
- 开工前必须完整读取 `doc/项目纪要-*.md`、`PRD详细版.md`、`技术方案.md`、`DESIGN.md`、`tokens.css`、`页面清单.md`、`pages/*.md`、`pages/*.html` 和本 `tests/` 目录。
- 文档完整性是硬门禁：页面清单中的 MVP 页面必须同时具备页面文档、HTML 原型和对应测试设计；缺失时先补文档，不能凭剩余文档开工。
- `pages/*.html` 是视觉、布局、组件和交互蓝图。实现每个页面前必须逐页读取对应 `.md` 与 `.html`，页面结构、按钮数量、表格列、组件关系和交互状态必须对齐；原型中的示例人名、数字和列表数据必须替换为后端真实返回。
- 单页开发闭环：读页面文档和原型 -> 1:1 实现结构并接真实数据 -> 按本 tests 用例转绿 -> 浏览器验证页面可打开、交互真实生效、无控制台错误、桌面和移动布局无破坏。
- 工程健壮性红线：列表必须分页并避免 N+1；登录、反馈链接、AI 调用和上传必须限流或校验；长任务进入 worker 并具备超时、重试和崩溃恢复；语义层配置缺失必须抛错；外部内容进入 LLM 必须作为数据隔离，不能当指令执行；多表状态变更必须事务化并支持幂等。

## 1. 完成定义
- 交付门：`测试索引.md` 中全部 P0 + P1 用例转绿，共 88 条。
- P2：尽力完成，不阻塞 MVP 交付；P3：本轮无。
- 安全鉴权层特例：全部 P0 零失败，不可降级；出现销售越权、配置越权、反馈链接绕过、危险上传、未授权社媒抓取任一失败即未交付。
- 覆盖率门槛：业务核心模块 >= 90%；基础设施模块 >= 80%；UI 胶水层 >= 60%。
- 测试来源：只能从 PRD、技术方案、设计决策蓝图、页面文档和本 tests 目录派生，不得从实现代码反推需求。

## 2.1 逐页 TDD 硬门禁
- `tests/阶段1_*_TDD.md` 是阶段一 MVP 的逐页开发入口，数量必须与 `pages/阶段1_*.md` 完全一致。
- Coding Agent 实现任一页面前，必须同时读取该页 `pages/<页面名>.md`、`pages/<页面名>.html`、`tests/<页面名>_TDD.md`，再结合上表分层 TDD 补齐接口、安全、边界、无障碍与性能测试。
- 逐页 TDD 优先证明页面结构、真实 API、持久化读写、权限边界和原型一致性；分层 TDD 负责跨页面业务闭环与横切风险。两者冲突时，以 PRD/技术方案/DESIGN 为准并记录纪要。

## 4. Coding Agent 消费协议
1. 先读取本契约的完成定义，再读取 `测试索引.md`。
2. 按 P0 -> P1 -> P2 的顺序选择用例；同一优先级内优先安全、数据隔离、线索闭环、客户背景调查、Banner、再营销人工确认。
3. 对每条用例执行四步：
   - 写测试并运行，确认失败方式与“预期失败原因”一致。
   - 写最小实现并运行，确认该用例转绿。
   - 回退实现或故意破坏关键逻辑并运行，确认测试重新失败，证明不是假绿。
   - 恢复实现并运行相关层回归。
4. 每转绿一条，在 `测试索引.md` 的状态列或单独进度文件中记录。
5. 每完成一层，运行该层全量回归；跨层改动必须补跑受影响层。
6. 全部 P0 + P1 转绿，且覆盖率门槛满足后，才算交付。

## 6. 不允许的交付捷径
- 不允许只隐藏前端入口来替代接口权限控制。
- 不允许 mock 被测领域逻辑本身。
- 不允许让再营销邮件绕过人工确认。
- 不允许让 Facebook/领英来源变成未授权自动抓取。
- 不允许把客户背景调查人工修改覆盖自动原文。
- 不允许把 Banner、来源字典、账号权限的配置变更漏掉审计。
- 不允许用 mock、硬编码、静态 JSON 或原型示例数据冒充真实后端读写。
- 不允许跳过失败测试继续堆新功能。
- 不允许用 `any`、`eslint-disable`、`@ts-ignore`、弱断言或“未抛异常即通过”绕过问题。
