# 测试决策蓝图

## 1. 测试范围
- 选中层级：全 8 层，包括单元、组件、集成契约、E2E、安全鉴权、边界异常、无障碍、性能。
- 阶段边界：默认覆盖 MVP 阶段，同时把已进入页面和文档的阶段2能力纳入门禁，包括再营销提示词/附件、客户态势和配置页。
- 重点包：线索进入、来源字典、客户背景调查、客户详情、账号权限、销售数据隔离、全局 Banner、再营销下一步动作、提示词/附件、大模型草稿、人工确认发送和审计留痕。

## 2. 各层测试重点
- 单元层：测试来源字典、客户背景调查版本、销售数据范围、Banner 发布校验、再营销上下文拼装和禁用词校验等纯业务规则。
- 组件层：测试登录页、工作台、线索池、客户详情、设置管理、账号权限、再营销待办和草稿详情页面的渲染、表单状态、只读/无权限状态和关键交互。
- 集成契约层：测试 Auth、Inquiry、SourceDictionary、CustomerBackgroundInvestigation、BannerConfig、NurtureTask、Attachment、AuditLog 等接口 schema、状态码和数据库副作用。
- E2E 层：测试从线索进入到销售反馈、客户池沉淀、背景调查、再营销草稿、人工确认发送的完整闭环，以及管理员配置来源/Banner/账号权限后对前台页面生效。
- 安全鉴权层：测试登录会话、RBAC、销售水平越权、垂直越权、反馈链接 7 天有效、导出脱敏、配置审计和上传安全。
- 边界异常层：测试缺字段、重复提交、任务失败、网络超时、超大导入、非法来源状态迁移、Banner 缺失、附件超限等异常。
- 无障碍层：测试后台关键页面的键盘可达、焦点顺序、表单标签、按钮语义、Banner 可读、错误提示可被读屏获取。
- 性能层：测试列表查询、导入任务、背景调查异步任务、再营销生成、报表聚合和首屏加载的 MVP 基线。

## 3. 验收门禁
- 完成定义：`output/tests/测试索引.md` 中全部 P0 + P1 用例转绿才算交付；P2 尽力完成；P3 可延期。
- 安全层特例：安全鉴权层 P0 零失败，不可降级。
- 覆盖率目标：业务核心模块 >= 90%；基础设施模块 >= 80%；UI 胶水层 >= 60%。
- 变异测试：MVP 阶段不强制 mutation score，但核心纯函数单元测试必须覆盖有效/无效分区和边界两侧。
