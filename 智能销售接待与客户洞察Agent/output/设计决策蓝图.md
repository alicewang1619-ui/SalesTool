# 设计决策蓝图（阶段 I 进行中）

## 1 盘子

- 产品定位：内部使用的 B2B 销售增长系统，不做客户登录门户。
- 端形态：PC 管理后台 + 手机销售反馈卡片。
- 响应式策略：PC 管理后台以桌面宽屏为主；手机端只承载销售反馈卡片，避免把复杂后台完整塞进手机。
- 工作台聚合：今日询盘、有效线索、未反馈、待确认再营销草稿、高意向客户。

## 2 信息架构 & 站点地图

- 导航模式：PC 后台使用左侧常驻导航；详情页保留面包屑；销售端反馈卡片不进入完整后台导航。
- 主分区：
  - 工作台：承载每日总览、异常提醒和待办入口。
  - 线索池：管理询盘进入、AI 接待补全、客户画像评分、国家分发和线索详情。
  - 客户池：沉淀高意向客户、有效客户、已报价未回复、资料库客户、已转代理商、无效、撤单/流失和已成交客户。
  - 报表中心：查看日报、月报、季报、年报以及渠道、区域、产品、销售反馈等非金额指标。
  - 再营销待办：查看 AI 生成的再营销建议、邮件草稿、触达理由和人工确认发送队列。
  - 系统配置：维护渠道接入、国家/区域/销售映射、产品字典、评分规则和账号权限。
- 页面调用关系骨架：
  - 工作台 -> 线索池/线索详情/未反馈列表/再营销待办/报表中心。
  - 线索池 -> 线索详情 -> 分发记录 -> 销售反馈卡片。
  - 线索详情 -> 客户详情/客户池记录/再营销草稿。
  - 客户池 -> 客户详情 -> 再营销待办/报表中心。
  - 报表中心 -> 日报/月报/季报/年报 -> 指标明细。
  - 系统配置 -> 渠道、区域销售映射、产品字典、评分规则、账号权限。

## 3 角色 & 权限

- 账号体系：PC 后台使用自建账号密码；销售反馈卡片使用带过期时间的安全链接，第一版不要求销售进入完整后台。
- 角色分层：
  - 管理员：维护系统配置、账号权限、渠道导入配置、国家/区域/销售映射、产品字典、评分规则、AI 接待 playbook 和提醒规则。
  - 营销/运营：使用工作台、线索池、客户池、报表中心和再营销待办；可查看全局线索、调整客户状态、确认再营销草稿、导出报表和催促未反馈。
  - 销售：只查看分配给自己的线索摘要，通过手机反馈卡片提交已联系、已报价、无效、需跟进、已成交等状态和简短客户判断；不进入复杂 PC 后台。
- 同页角色差异：
  - 无权限的一级导航隐藏。
  - 页面内不可执行动作置灰，并给出简短原因。
  - 非负责人打开销售反馈卡片时只读或提示无权限。
  - 管理员配置变更必须记录操作人和时间。
