# 方案 V0

## 1. 方案目标

把当前分散在官网在线聊天、WhatsApp、邮箱、Facebook、LinkedIn、Google 询盘、官网后台、展会名片、Facebook Message 和 LinkedIn 收件箱里的 ultrasound 海外询盘，统一沉淀为可接待、可补全、可评分、可分发、可反馈、可分析、可再营销的客户资产。

第一版不做重 CRM，不做成交金额/报价金额统计，不做未经确认的自动群发。系统核心是降低响应延迟、减少人工整理、补齐客户信息、推动销售反馈，并让营销负责人看到客户池和渠道质量。

## 2. V0 产品结构

### M1. 多渠道询盘收口

**目标**: 先把询盘变成统一线索对象。

第一版支持：

- 官网在线聊天
- 官网后台询盘
- 邮箱询盘
- WhatsApp 询盘
- Facebook Message 聊天记录
- LinkedIn 收件箱
- Google 询盘
- 展会名片手动录入/批量导入

第一版可接受“自动接入 + 半自动导入”混合模式。重点不是一开始把所有渠道 API 全打通，而是先让所有线索进入统一池。

### M2. AI 在线接待与 ultrasound 引导提问

**目标**: 客户在线咨询时，Agent 先接住，并把模糊询盘问成有效询盘。

核心提问顺序：

1. 识别身份：Company / Distributor / Hospital / Clinic / Doctor / Other
2. 识别国家：用于区域销售分发
3. 识别应用：Radiology / Cardiology / OB/GYN / Emergency / Veterinary / Other
4. 识别产品类型：Trolley / Portable / Handheld
5. 识别具体需求：用途、预算、数量、采购时间、是否需要报价/资料/演示

原则：

- 不立刻给泛泛报价，先补齐身份、应用和需求。
- 复杂问题交给销售，例如代理政策、证书承诺、售后承诺、特殊价格。
- AI 不能冒充销售承诺价格、独家代理或注册证书。

### M3. 双客群客户画像

**目标**: 代理商和医生/终端用户走不同画像字段。

代理商画像：

- 重点国家
- 历史合作业务，例如 IVD、MR、影像、耗材
- 代理过的品牌
- 是否已有 ultrasound 业务
- ultrasound 涉及品牌
- 年业务规模

医生/终端用户画像：

- 所属科室
- 临床应用
- 之前用过的机器
- 预算
- 想购买的产品
- 应用场景

### M4. 客户背景调查

**目标**: 减少人工查背景，让销售拿到更完整上下文。

第一版数据源：

- 客户官网
- 邮箱域名
- 公开网页搜索
- LinkedIn/Facebook 公开信息或授权数据
- 历史对话与销售反馈

输出：

- 公司/机构简介
- 国家/区域
- 客户类型判断
- 可能业务方向
- 与 ultrasound 相关的需求线索
- 背景信息可信度标记

### M5. 线索评分与优先级

**目标**: 不用金额判断线索价值，而用完整度和销售价值判断。

硬门槛：

- 身份明确
- 应用明确
- 需求明确

评分因素：

- 国家/区域是否为重点市场
- 客户类型：代理商、医生、医院、诊所等
- 产品类型：Trolley / Portable / Handheld / 具体型号
- 是否明确预算
- 是否明确采购时间
- 是否有历史设备/品牌经验
- 是否有真实业务场景
- 是否已经提供联系方式
- 渠道来源质量

输出等级：

- 高意向客户
- 有效客户，持续跟进
- 待补充信息
- 暂无明确需求，放入资料库
- 无效客户

### M6. 国家/区域销售分发

**目标**: 按国家/区域自动分给对应销售。

流程：

1. 先识别客户国家
2. 国家映射到区域，例如拉美、北美、中东
3. 区域/国家映射到销售负责人
4. 生成销售线索卡片
5. 通过微信/邮件发送给销售

异常：

- 国家不明确：Agent 优先追问 “Which country are you from?”
- 国家无法判断：进入待分配池，由负责人手动分配
- 销售负责人缺失：提醒管理员补充国家/销售映射表

### M7. 轻量销售反馈卡片

**目标**: 不做重 CRM，让销售 10-30 秒完成反馈。

销售收到的信息：

- 客户摘要
- 国家/区域
- 客户类型
- 应用领域
- 需求摘要
- AI 线索等级
- 建议跟进动作

销售只需反馈：

- 状态：已联系 / 已报价 / 无效 / 需跟进 / 已成交
- 客户判断：有效客户继续跟进 / 放入资料库后续再跟进 / 确认真实需求 / 已转给代理商跟进
- 可选备注：一句话即可

提醒规则建议：

- 24 小时未反馈：提醒销售
- 48 小时未反馈：进入管理者关注
- 高意向客户未反馈：优先提醒

### M8. 客户分析报表体系

**目标**: 从日报到月报、季报、年报，统一看非金额型经营数据。

指标：

- 询盘总量
- 有效数量
- 国家分布
- 渠道质量
- 代理商占比
- 医生占比
- 产品分类
- 销售反馈
- 未及时反馈情况
- 撤单情况
- 再营销数据
- 客户池变化

明确不做：

- 成交金额统计
- 报价金额统计
- 销售业绩金额归因

### M9. 客户池与再营销

**目标**: 把未成交客户沉淀成可经营资产。

客户池分类：

- 高意向客户
- 有效客户，持续跟进
- 已报价未回复
- 暂无明确需求，放入资料库
- 已转代理商跟进
- 无效客户
- 撤单/流失客户
- 已成交客户

再营销规则：

- AI 生成触达建议、邮件草稿、触达理由和推荐时间。
- 第一版必须人工确认后发送。
- 不做未经确认的自动群发。

### M10. 客户态势监督与商机发现

**目标**: 不是做社媒监听，而是做客户背景调查、商机发现和再营销触发。

第一版轻量关注：

- 客户官网公开更新
- LinkedIn/Facebook 公开动态或授权数据
- 展会、开业、扩张、采购、设备更新、团队扩张等信号
- 邮件打开、点击、重新访问等历史行为
- 销售反馈中标记的后续机会

约束：

- LinkedIn/Facebook 不做违规抓取。
- 先用公开数据、官方 API、用户授权数据和人工导入。
- 商机触发先生成提醒和草稿，不自动发送。

## 3. 第一版 MVP

第一版先做 6 件核心事：

1. 多渠道询盘统一进入线索池
2. AI 接待与 ultrasound 引导提问
3. 客户背景调查与双客群画像
4. 线索评分 + 国家/区域销售分发
5. 轻量销售反馈卡片 + 未反馈提醒
6. 日报/月报所需的客户分析报表

第一版轻量做：

- 客户池分层
- 再营销草稿生成
- 客户态势监督信号记录

第一版不做：

- 重 CRM
- 金额/报价统计
- 全渠道完全自动 API 接入
- 自动群发再营销
- 未授权社媒抓取

## 4. 关键流程

### 流程 A：客户在线咨询

1. 客户发起咨询，例如 “I want to buy an ultrasound.”
2. Agent 回复并引导补齐身份、国家、应用、产品类型和需求。
3. Agent 识别客户是代理商还是医生/终端用户。
4. Agent 根据身份切换画像字段。
5. 系统生成线索摘要和评分。
6. 国家明确后自动分发给对应销售。
7. 销售收到轻量反馈卡片。
8. 销售反馈状态和判断。
9. 系统进入报表和客户池。

### 流程 B：客户信息不完整

1. 客户只问价格或只说 “Hi”。
2. Agent 先礼貌接待，再追问必要字段。
3. 3 轮内尽量补齐身份、国家、应用、需求。
4. 补不齐则标记为“待补充信息”或“低优先级培育”。
5. 必要时交给人工查看。

### 流程 C：销售未反馈

1. 线索分发后开始计时。
2. 24 小时无反馈，提醒销售。
3. 48 小时无反馈，进入管理者关注。
4. 高意向客户未反馈，优先提醒。
5. 统计到报表的“未及时反馈情况”。

### 流程 D：再营销

1. 客户进入客户池。
2. 系统根据状态生成再营销建议。
3. AI 生成邮件草稿和触达理由。
4. 用户或销售人工确认。
5. 发送后记录结果，进入再营销数据和客户池变化。

## 5. 核心数据对象

| 对象 | 说明 |
|---|---|
| Inquiry | 原始询盘，记录来源、时间、渠道、原始内容 |
| Conversation | 对话记录，记录 Agent 与客户沟通过程 |
| CustomerProfile | 客户画像，区分代理商画像和医生/终端用户画像 |
| LeadScore | 线索评分，记录完整度、优先级、评分原因 |
| SalesAssignment | 销售分发记录，记录国家、区域、销售负责人 |
| SalesFeedback | 销售反馈，记录状态、客户判断、备注、反馈时间 |
| ReportMetric | 报表指标，支持日报、月报、季报、年报汇总 |
| CustomerPool | 客户池状态，记录客户分层和后续动作 |
| NurtureTask | 再营销任务，记录草稿、触达理由、确认状态 |
| CustomerSignal | 客户态势信号，记录官网/社媒/邮件/销售反馈等机会信号 |

## 6. 关键决策点

### 已确定

- 产品是 ultrasound，含 Trolley、Portable、Handheld。
- 客户包括代理商和医生/终端用户。
- 高质量线索标准是身份明确、应用明确、需求明确。
- 销售分发按国家/区域。
- 销售反馈必须轻量，不能做重 CRM。
- 报表不统计金额。
- 再营销必须人工确认后发送。

### 待拍板

- 第一版哪些渠道必须自动接入，哪些允许手动/半自动导入。
- 微信/邮件反馈卡片的具体承载方式。
- 24 小时/48 小时未反馈提醒是否合适。
- 各区域/国家与销售负责人映射表谁维护。
- 产品型号清单和 ultrasound 知识库由谁提供。
- 再营销邮件使用哪个发送工具，退订机制如何做。
- 客户态势监督第一版到底接哪些数据源。

## 7. V0 判断

这版方案的核心不是“做一个 CRM”，而是先打通询盘到销售反馈的闭环：

> 多渠道询盘进入统一池，AI 把模糊咨询问清楚，系统按国家/区域分给销售，销售用 10-30 秒反馈，管理者每天看到线索质量和客户池变化。

只要这个闭环跑通，后面的客户态势监督、再营销、月报季报年报和 CRM 集成才有真实数据基础。
