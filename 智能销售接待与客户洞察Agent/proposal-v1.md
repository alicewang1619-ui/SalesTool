# 方案 V1

## 1. V1 核心收敛

V1 从“AI 客户增长中台”收敛为第一版可落地闭环：

> 先把每月 300+ 条 ultrasound 询盘从分散入口拉进统一线索池，AI 负责接待和补齐关键信息，系统按国家/区域分给销售，销售用 10-30 秒反馈，管理者每天看到询盘质量、销售反馈和客户池变化。

第一版不追求全自动、全渠道、全 CRM，而是优先证明三件事：

1. 客户响应更快。
2. 销售反馈回来。
3. 管理者看得见漏斗漏在哪里。

## 2. MVP 范围

### 必须做

1. **统一线索池**
   - 汇总官网聊天、官网后台、邮箱和展会名片导入。
   - WhatsApp、Facebook、Google 可作为 P1 接入。
   - LinkedIn 收件箱和深度社媒监测后置。

2. **AI 接待与 ultrasound 引导提问**
   - 用 ultrasound 专属 playbook 补齐身份、国家、应用、产品类型、具体需求。
   - 支持代理商和医生/终端用户两套提问分支。
   - 复杂价格、代理政策、证书、售后承诺必须升级销售。

3. **客户画像与背景摘要**
   - 代理商画像：国家、历史业务、代理品牌、超声经验、年业务规模。
   - 医生画像：科室、临床应用、用过的机器、预算、目标产品和场景。
   - 每条背景结论需要来源或可信度标记。

4. **线索评分**
   - 硬门槛：身份明确、应用明确、需求明确。
   - 输出：高意向客户、有效客户、待补充信息、资料库客户、无效客户。
   - 评分理由必须展示给销售，不能只给一个分数。

5. **国家/区域销售分发**
   - 维护国家/区域/销售映射表。
   - 国家明确后自动分发。
   - 国家不明确时，Agent 优先追问国家。
   - 无法判断时进入待分配池。

6. **轻量销售反馈卡片**
   - 通过微信/邮件发送线索摘要和反馈链接。
   - 销售只需选择状态和客户判断。
   - 状态：已联系、已报价、无效、需跟进、已成交。
   - 客户判断：有效客户继续跟进、放入资料库、确认真实需求、已转代理商跟进。
   - 备注可选。
   - 反馈目标：10-30 秒完成。

7. **未反馈提醒**
   - 24 小时未反馈：提醒销售。
   - 48 小时未反馈：进入管理者关注。
   - 高意向客户未反馈：优先提醒。

8. **非金额型报表**
   - 询盘总量
   - 有效数量
   - 国家分布
   - 渠道质量
   - 代理商/医生占比
   - 产品分类
   - 销售反馈
   - 未及时反馈
   - 撤单情况
   - 再营销数据
   - 客户池变化

9. **客户池基础分层**
   - 高意向客户
   - 有效客户，持续跟进
   - 已报价未回复
   - 暂无明确需求，放入资料库
   - 已转代理商跟进
   - 无效客户
   - 撤单/流失客户
   - 已成交客户

### 轻量做

1. **再营销草稿**
   - AI 生成邮件草稿、触达理由、推荐时间。
   - 必须人工确认后发送。
   - 先做“待确认发送”队列，不做自动群发。

2. **客户态势信号记录**
   - 先记录销售反馈、客户官网公开信息、邮件互动等轻量信号。
   - LinkedIn/Facebook 只使用公开/授权/API 数据。
   - 第一版不做深度监测系统。

### 后置

- 完整 CRM
- 金额/报价/业绩分析
- 全渠道自动 API 接入
- LinkedIn/Facebook 私域自动抓取
- 自动群发再营销
- 完整客户态势监督
- 企业微信深度集成
- 商业数据源集成

## 3. 用户角色

| 角色 | 需要完成什么 |
|---|---|
| 营销负责人 | 看询盘质量、渠道效果、国家分布、销售反馈、客户池变化 |
| 销售 | 快速理解客户、联系客户、用最少动作反馈判断 |
| 管理员 | 维护产品知识库、国家/销售映射、渠道接入、反馈提醒规则 |
| Agent | 接待客户、补齐字段、查背景、评分、生成摘要、提醒和报表 |

## 4. 关键体验

### 销售收到的线索卡片

必须一屏看完：

- 客户是谁
- 来自哪个国家/区域
- 代理商还是医生/终端用户
- 关注哪类 ultrasound
- 应用场景是什么
- 为什么系统判断为高意向/有效/待补充
- 建议下一步怎么跟

销售反馈：

- 第一步：选状态
- 第二步：选客户判断
- 第三步：可选备注

### 管理者日报

日报只看核心和异常：

- 今日总询盘 / 有效数量
- 高意向客户清单
- 国家/渠道/产品概览
- 未及时反馈销售和线索
- 撤单/流失提醒
- 需要人工确认的再营销草稿

月报/季报/年报再看趋势。

## 5. 数据与合规原则

1. 所有客户背景结论必须保留来源或可信度。
2. LinkedIn/Facebook 不做未授权抓取。
3. 再营销必须人工确认后发送。
4. 客户数据需要有权限控制。
5. 销售反馈链接需要过期时间和身份标记。
6. AI 不承诺价格、独家代理、注册证书、售后政策。

## 6. MVP 成功指标

| 指标 | 当前基线 | MVP 目标 |
|---|---:|---:|
| 月询盘处理量 | 300+ | 300+ 全部进入统一线索池 |
| 单条人工初步处理 | 约 10 分钟 | 显著减少人工整理与分发时间 |
| 客户损失 | 20%-30% | 通过即时接待和快速分发逐步降低 |
| 销售无反馈 | 50%+ | 通过轻量反馈卡片和提醒逐步降低 |
| 销售反馈耗时 | 未标准化 | 10-30 秒完成 |
| 国家/区域分发 | 人工判断 | 国家明确后自动分发 |
| 报表 | 手工/分散 | 日报自动生成，月/季/年报有数据基础 |

## 7. 关键待确认问题

1. 第一版 P0 渠道是否确认：官网聊天、官网后台、邮箱、展会名片导入。
2. WhatsApp、Facebook、Google 询盘是否作为 P1 接入。
3. 销售反馈链接通过微信还是邮件优先发送。
4. 国家/区域/销售映射表由谁维护。
5. 产品型号和知识库由谁提供。
6. 24 小时/48 小时未反馈提醒是否合适。
7. 再营销邮件发送工具和退订机制如何做。

## MVP 划分（已与企业家确认）

### 第一版必须做

- 统一线索池：官网聊天、官网后台、邮箱、展会名片导入。
- AI 接待与 ultrasound 引导提问。
- 代理商/医生双客群画像。
- 客户背景摘要和线索评分。
- 国家/区域销售分发。
- 轻量销售反馈卡片。
- 未反馈提醒。
- 非金额型基础报表。
- 客户池基础分层。

### 第一版轻量做

- 再营销草稿，人工确认后发送。
- 客户态势信号记录。

### 明确后置

- WhatsApp、Facebook、Google 询盘的深度自动接入。
- 完整 CRM。
- 成交金额/报价金额分析。
- LinkedIn 收件箱和深度社媒监测。
- LinkedIn/Facebook 私域自动抓取。
- 自动群发再营销。
- 企业微信/CRM 深度集成。

### MVP 完成定义

完成统一线索池、AI 接待引导、客户画像评分、国家分发、销售轻反馈和基础报表，即第一版 MVP 完成。

## 8. V1 判断

V1 方向比 V0 更收敛：

> 第一版不是做“完整客户增长中台”，而是做“询盘闭环 MVP”：统一线索池、AI 接待、客户画像、国家分发、轻量反馈、非金额型报表。只要闭环跑起来，客户池、再营销和态势监督才有真实数据可用。
