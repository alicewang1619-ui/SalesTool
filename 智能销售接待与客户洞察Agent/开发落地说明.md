# 开发落地说明

## 当前生产化骨架
- `backend/`：FastAPI + SQLAlchemy，已接真实数据库读写；本地默认 SQLite，Docker Compose 使用 Postgres。
- `frontend/`：React + Vite + Ant Design，已接登录、全局 Banner、工作台、线索池、客户详情、设置管理首批路由。
- `docker-compose.yml`：提供 Postgres、Redis、API、前端容器编排入口；worker/scheduler 将在客户背景调查、再营销草稿、邮件发送等长任务落地时补齐。

## 与规格的关系
- 开发前必须继续执行 `output/tests/验收门禁与GOAL契约.md`。
- 阶段一 18 个 MVP 页面已具备逐页 TDD：`output/tests/阶段1_*_TDD.md`。
- 当前骨架只完成生产应用的第一批真实 API 与首批前端路由，不代表 18 个 MVP 页面全部完成；后续必须按“单页开发闭环”逐页复刻 `pages/*.html` 并接真实 API。


## 阶段一页面闭环进度
- `阶段1_账号权限_登录页` 已进入 GOAL 单页闭环：前端按原型补齐左侧导航、全局 Banner、双栏登录表单、账号输入、密码输入、登录按钮、页面内错误提示和会话过期提示；原型示例账号密码不作为表单真实预填数据。
- 登录页已接真实后端：`POST /api/auth/login` 返回 JWT、角色和姓名；`GET /api/me` 用于后台壳层会话校验；未登录或失效 token 访问 `/admin/*` 会回到登录页并展示“会话已过期，请重新登录。”。
- 登录入口已补生产底线：连续失败写入 `login_attempts` 持久化表，5 次失败后锁定 15 分钟；每次登录请求写入审计日志并返回 `x-trace-id`；管理员可通过 `GET /api/audit-logs` 查询分页审计记录。
- 本页验证结果：`py -m pytest .\tests -q` 为 7 passed；`npm.cmd run build` 通过；Playwright 浏览器验收确认登录页存在 1 个 Banner、6 个侧栏项、1 个账号输入、1 个密码输入，错误登录返回真实 401，正确登录跳转工作台并写入 token，清空 token 后访问后台会回登录页。
